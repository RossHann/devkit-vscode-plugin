Dedicated Software Porting

The tool provides source code porting of commonly used big data software. The tool can automatically modify the source code, compile the code, and build the software packages compatible with the Kunpeng platform. This function is available only on the Kunpeng platform and supports only CentOS 7.6.

Prerequisites

You have logged in to the Kunpeng Porting Advisor WebUI.

The required compiler tool has been installed in the /usr/bin or /usr/local/bin directory.

Procedure

  1. Choose Dedicated Software Porting from the navigation pane on the left.

    On the Before You Start dialog box displayed, select I have read the above and click OK. The page shown in Figure 1 is displayed.

  2. Select the Kunpeng solution from the solution list on the left and click the software name of the specified version from the software list on the right.

    The dedicated software porting details are displayed, as shown in Figure 2. Table 2 describes the parameters.

    All solution-specific software porting templates supported are stored in the /opt/portadv/resource/migration directory on the server. (/opt/portadv is the tool installation directory. Replace it with the actual directory.) All users can use the templates in this directory.

  3. In the Perform Step areas, select the steps to be performed and click Start.

    • The operations in the Check Environment area are automatically executed by default.
    • After you click Start, the execution status will be displayed before the selected steps to be executed.
    • The tool automatically modifies, compiles, and builds your software packages.
    • During dedicated software porting, you can leave the current page and perform other tasks, excluding restoring and updating software porting templates.
    • To cancel a task, click Close during the task running process.
    • The workspace for dedicated software porting is /opt/portadv/xx/migration. xx indicates the user name, and /opt/portadv is the tool installation directory. Replace it with the actual directory. All operations and result files are stored in this workspace.
    • By default, logs of the dedicated software porting are stored in the /opt/portadv/logs/porting.log file. /opt/portadv is the tool installation directory. Replace it with the actual directory. You can view the failure cause in the log file and rectify the fault based on the suggestions.

  4. Click Download Porting Result to download the software files ported.
