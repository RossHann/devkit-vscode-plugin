CRL Management

The certificate revocation list (CRL) is used to check the certificate validity of the server where the dependency files are downloaded during software package reconstruction.

  • A maximum of three certificates can be imported.
  • The CRL check function is enabled by default. To disable this function, choose > System Settings in the upper right corner of the page and modify the setting of Certificate Validity Check.

Prerequisites

You have logged in to the Kunpeng Porting Advisor.

Importing a CRL File

  1. In the upper right corner of the page, choose > Certificate Revocation List.

    The Certificate Revocation List page is displayed.
    Figure 1 Certificate Revocation List page

  2. Click Import.
  3. Select a CRL file and click Open.

    The imported CRL file is displayed. See Figure 2.

    Figure 2 Imported CRL file
    • If a direction arrow is displayed on the left of the file name, click the arrow to expand or collapse the certificate revocation list.
    • If the status column indicates that the certificate has expired, import the latest certificate as soon as possible.

Deleting a CRL File

  1. In the upper right corner of the page, choose > Certificate Revocation List.

    The Certificate Revocation List page is displayed.

  2. Locate the CRL to be deleted and click Delete.

    The Delete File dialog box is displayed.

    Figure 3 Deleting a file

  3. Click Confirm.
