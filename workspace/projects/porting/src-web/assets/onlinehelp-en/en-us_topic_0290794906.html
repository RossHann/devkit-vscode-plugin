Uploading Files

If the size of the file to be uploaded is greater than 1 GB or the size of the file after decompression is greater than half of the remaining drive space, you need to free up some drive space or manually upload the file to the server. In other cases, you can use the upload function on the WebUI.

In the following example, /opt/portadv is the tool installation directory. Replace it with the actual directory.

  1. Obtain the files to be uploaded.
  2. Place the file to be uploaded in the subdirectory of the created user name directory. The following describes how to use the portadmin user to upload the source code porting file. For details about the directory, see the upload directory on the page.

    1. Use an FTP tool (for example, WinSCP) to log in to the server OS as the root user.
    2. Upload the source code file to be scanned to the /opt/portadv/portadmin/sourcecode/ directory.
    3. Decompress the package. If the file does not need to be decompressed, skip this step.
      1. Use SSH to log in to the OS CLI as the root user.
      2. Run the following command to switch to the directory where the file is located.
        1
        cd /opt/portadv/portadmin/sourcecode
      3. Decompress the file. For example, if the file is in tar.gz format, run the following command:
        1
        tar -zxvf xxx.tar.gz
    4. Change the owner group of the file and change the execute permission on the file. (xxx indicates the folder where the package is extracted or the software installation package uploaded by the user.)
      chown -R porting:porting xxx
chmod -R 600 xxx

      If you want to use the chmod command instead of the chown command, run the following command:

      1
      chmod -R o+rw xxx

After the file is uploaded, click the text box on the WebUI. A drop-down list box is displayed. The drop-down list box displays the uploaded files (up to 50 files). You can select the file to be scanned, or you can enter the file name.
