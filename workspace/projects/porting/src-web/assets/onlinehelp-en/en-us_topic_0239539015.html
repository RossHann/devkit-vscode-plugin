Run Log Management

Prerequisites

You have logged in to the Kunpeng Porting Advisor WebUI.

  • Only the administrator portadmin can view and download run logs and change the run log level.
  • The administrator can click and choose System Settings to set the log level. The logs of the specified level and higher levels will be recorded.

Procedure

  1. On the WebUI, click in the upper right corner and choose Log.

    On the Log page displayed, choose Run Log, as shown in Figure 1.
    Figure 1 Run log

  2. Click One-Click Download to download run logs.

    The run logs downloaded are in the following format:

    [Time] [Log level] [Process ID] [Thread name] [Module name: Function name: Number of lines] Log details

    Example:

    [2020-08-20 09:50:21] [INFO] [processID:18508] [MainThread] [solution_api:get_solution_category:143] get_solution_category
    The log level indicates the importance of the log information. The default log level is INFO. The tool run logs are classified into the following levels:
    • DEBUG: debugging information for fault locating.
    • INFO: key information about the normal running of the service.
    • WARNING: events about the system in unexpected status that does not affect the running of the system.
    • ERROR: errors that do not affect the application running.
