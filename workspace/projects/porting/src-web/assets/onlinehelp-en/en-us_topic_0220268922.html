Updating a Software Porting Template

Prerequisites

  • You have logged in to the Kunpeng Porting Advisor WebUI.
  • You have obtained the latest software porting template.

    The software porting template in versions earlier than 2.3.T10 cannot be upgraded to the software porting template of version 2.3.T10.

Procedure

  1. On the WebUI, click in the upper right corner and choose Software Porting Template.
  2. Click Upgrade. The page shown in Figure 1 is displayed.

    Figure 1 Updating the software porting template

  3. Click Select File and select the software porting template package to be uploaded. If the verification is successful, click Upload, enter the password of the administrator, and click OK.

    The update progress is displayed. After the update is complete, check the software porting template file in the /opt/portadv/resource/migration directory.
    • Do not close the current page during the upgrade process.
    • After you select the software porting template resource package to be uploaded, the tool checks whether the resource package exists on the server. If yes, you need to delete it first. When the resource package is uploaded, the tool checks whether the current version is the latest. If yes, the update is not required.
    • If the upgrade fails, rectify the fault based on the error message displayed or by referring to the FAQs in the user guide. For details about the failure information, see /opt/portadv/logs/porting.log.
    • /opt/portadv is the tool installation directory. Replace it with the actual directory.
