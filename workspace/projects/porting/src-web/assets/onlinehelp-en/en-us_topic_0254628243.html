Log Description

Operation Log Description

Operation logs record information about the modification operations performed by users and scheduled tasks of the system. The following operations are logged:

  • User login or logout
  • Uploading files
  • Creating a task
  • Create or delete a user.
  • Locking or unlocking a user
  • Modifying user configuration
  • Editing a password
  • Downloading/Deleting a report
  • Rebuilding a software package
  • Setting the maximum number of online users
  • Setting the run log level
  • Reading a disclaimer for the first time
  • Changing the historical report threshold
  • restoring or upgrading the dependency dictionary
  • restoring or upgrading the software porting template
  • Periodic deletion of operation logs
  • Checking certificate validity
  • Importing or replacing a certificate
  • Setting the certificate expiry alarm threshold
  • Managing the weak password dictionary
  • Updating the working key

Run Log Description

Run logs record necessary information during program running, helping R&D personnel or users quickly locate and rectify faults.

Log files contained in the log package downloaded are listed in the following table:
Table 1 Description of run log modules

Run Log File

Description

porting.log

Records necessary information during program running, helping R&D personnel or users quickly locate and rectify faults.

all_asm.log

Records necessary information when the pure assembly files are scanned and ported during the source code porting process.

inline_asm.log

Records necessary information when the inline assembly files are scanned and ported during the source code porting process.

makefile.log

Records necessary information when the make file is compiled.

cmakefile.log

Records necessary information when the cmake file is compiled.

weakconsistency.log

Records necessary information when the memory consistency check is performed.

prevision.log

Records necessary information during the preprocessing process.

default.log

Records the exception occurred when a run log summary fails to be generated.
NOTE:

When the run log file of each module reaches 100 MB, a backup will be generated and compressed in the /opt/portadv/logs directory. The /opt/portadv directory is the tool installation directory, which can be customized during tool installation. A maximum of two compressed packages can be stored on the server. When a new log package is generated, the earliest package will be deleted by default.
