Updating the Dependency Dictionary

Prerequisites

  • You have logged in to the Kunpeng Porting Advisor WebUI.
  • You have obtained the latest dependency dictionary package from the Kunpeng community.

    The dependency dictionary in versions earlier than 2.3.T10 cannot be upgraded to the dependency dictionary of version 2.3.T10.

Procedure

  1. In the upper right corner of the page, choose > Dependency Dictionary.
  2. Click Upgrade. The page shown in Figure 1 is displayed.

    Figure 1 Upgrading the dependency dictionary

  3. Click Select File and select the dependency dictionary to upload. After the file is verified, click Upload, enter the administrator password, and click OK.

    The upgrade progress is displayed. After the upgrade is complete, check the dependency dictionary in the /opt/portadv/config/dependency_dictionary directory.

    • Do not close the current page during the upgrade process.
    • When you are selecting the dependency dictionary to upload, the tool checks whether the selected package exists on the server. If it already exists on the server, delete it and then proceed subsequent operations. When the package is being uploaded, the tool checks whether the current whitelist is of the latest version. If yes, the file cannot be uploaded and no update is required.
    • If the upgrade fails, rectify the fault based on the error message displayed or by referring to the FAQs in the user guide. For details about the failure information, see /opt/portadv/logs/porting.log.
    • /opt/portadv is the tool installation directory. Replace it with the actual directory.
