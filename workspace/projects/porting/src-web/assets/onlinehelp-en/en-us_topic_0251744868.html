Configuring Alarm Thresholds for Historical Reports

Set the alarm thresholds for historical reports.

Prerequisites

You have logged in to the Kunpeng Porting Advisor WebUI.

Only the administrator portadmin can configure thresholds.

Procedure

  1. Click in the upper right corner and choose Threshold Settings.
  2. Click Modify, set Report Warning Threshold and Maximum Report Threshold, and click OK, as shown in Figure 1.

    Figure 1 Threshold Settings
    • The value of Report Warning Threshold ranges from 1 to 49, and the default value is 40. The value of Maximum Report Threshold ranges from 2 to 50, and the default value is 50. The value of Maximum Report Threshold must be greater than that of Report Warning Threshold.
    • When the number of reports reaches Report Warning Threshold, "Too many reports. Please delete some reports to release the storage space." will be displayed. However, the use of the tool is not affected.
    • When the number of historical reports reaches the Maximum Report Threshold, the task creating function is disabled, and the system displays a message indicating that the number of current reports has reached the maximum threshold and the task creating function has been disabled. In this case, you need to delete unnecessary historical reports.
