64-bit Mode Check

The tool identifies the 32-bit applications to be ported to the 64-bit application platform and provides modification suggestions. It supports GCC 4.8.5 to GCC 9.3.0.

Prerequisites

You have logged in to the Kunpeng Porting Advisor WebUI.
  • The build command to upload the build file in the source package is executed during the task running. Make sure that the content uploaded is the content that needs to be analyzed and checked and that there are no security issues to avoid affecting the normal operation of the server.
  • /opt/portadv is the default installation directory of the tool. The following uses this directory as an example. Replace it with the actual directory.

Procedure

  1. In the navigation pane on the left, select Enhanced Functions.

    Select 64-bit Running Mode, as shown in Figure 1.

    Figure 1 64-bit running mode check

  2. Set Source Code File Path. You can use either of the following methods:

    • Click Upload to upload the package or folder. The package will be automatically decompressed during the upload process.
      • Only tar, tar.bz, tar.bz2, tar.gz, tar.xz, tbz, tbz2, tgz, txz, and zip packages can be uploaded. Only one package can be uploaded at a time. The size of the source code package should be less than or equal to 1 GB. The decompressed file size should be less than or equal to half of the remaining drive space.
      • Only one folder can be uploaded at a time. The size of the folder must be less than or equal to half of the remaining drive space.
      • Internet Explorer is incompatible with the folder upload function. You need to use other browsers, such as Google Chrome and Microsoft Edge, to upload folders.
    • Manually upload the source code file to the specified path (for example, /opt/portadv/portadmin/precheck/) on the server. Grant the read, write, and execute permissions to the porting user. Then, click the text box to select the source code file path from the drop-down list. You can also manually enter the source code file path.

  3. Click Check. After the check is complete, click View Report. The page shown in Figure 2 is displayed.

    When editing the original code, save the modification in time. Otherwise, the modification may be lost due to session timeout.

    Figure 2 Check report
    Table 1 Parameter description

    Parameter

    Description

    File list

    Lists the source code files analyzed.

    Original Source Code

    Displays the original source code. You can also move the cursor to the code with a red wavy line and click Peek Problem in the floating message box to view the source code suggestions. You can modify the source code. After the modification, click Save, Undo, or Cancel in the upper right corner as required.
    • The tool supports concurrent running of multiple 64-bit mode checks.
    • To cancel a task, click Close during the task running process.
    • If the source code does not need to be modified, a dialog box is displayed indicating that the operation is successful. The check report page is not displayed.
    • To download a source code file, select the file on the left and click the download icon.
