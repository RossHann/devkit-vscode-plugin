Structure Byte Alignment Check

The tool checks the byte alignment of the structure variables in the source code.

Prerequisites

You have logged in to the Kunpeng Porting Advisor WebUI.
  • The build command to upload the build file in the source package is executed during the task running. Make sure that the content uploaded is the content that needs to be analyzed and checked and that there are no security issues to avoid affecting the normal operation of the server.
  • /opt/portadv is the default installation directory of the tool. The following uses this directory as an example. Replace it with the actual directory.

Procedure

  1. In the navigation pane on the left, select Enhanced Functions.

    Select Structure Byte Alignment as the check type, as shown in Figure 1.

    Figure 1 Structure byte alignment check
    Table 1 Parameter description

    Parameter

    Description

    Source Code File Path

    Enter a relative path by using either of the following methods:

    • Click Upload to upload the package or folder. The package will be automatically decompressed during the upload process.
    • Manually upload the source code file to the specified path (for example, /opt/portadv/portadmin/bytecheck/) on the server. Grant the read, write, and execute permissions to the porting user. Then, click the text box to select the source code file path from the drop-down list. You can also manually enter the source code file path.
    NOTE:
    • Only tar, tar.bz, tar.bz2, tar.gz, tar.xz, tbz, tbz2, tgz, txz, and zip packages can be uploaded. Only one package can be uploaded at a time. The size of the source code package should be less than or equal to 1 GB. The decompressed file size should be less than or equal to half of the remaining drive space.
    • Only one folder can be uploaded at a time. The size of the folder must be less than or equal to half of the remaining drive space.
    • Internet Explorer is incompatible with the folder upload function. You need to use other browsers, such as Google Chrome and Microsoft Edge, to upload folders.

    Build Tool

    Select the build tool to be used.

    Value:

    • make
    • cmake
    • automake

    Compile Command

    Specify the command used to compile the source code.

    The compile command is determined based on the configuration file of the build tool. For details, see Identifying Compile Commands Using the Build Tool Configuration File.

  2. Click Check to start the byte alignment check. After the check is complete, click View Report. The page shown in Figure 2 is displayed.

    When editing the original code, save the modification in time. Otherwise, the modification may be lost due to session timeout.

    Figure 2 Alignment check
    Table 2 Parameter description

    Parameter

    Description

    File list

    Lists the source code files analyzed.

    Original Source Code

    Displays the original source code.

    Structure Variable Memory Allocation

    Displays the memory space to be aligned in the code.
    • The tool supports concurrent running of multiple alignment checks.
    • To cancel a task, click Close during the task running process.
    • Only one of the 64-bit mode check and structure type definition alignment check can be performed by a user at a time.
    • You can click the arrow keys in the upper right corner of Original Source Code to view the code.
    • When the environment changes, the analysis result may be different, which is normal.
