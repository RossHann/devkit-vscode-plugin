Web Server Certificate Management

The web server certificate is used for communication between the Kunpeng Hyper Tuner web server and web browser.

Prerequisites

You have logged in to the Kunpeng Hyper Tuner.

The administrator (tunadmin) can generate certificate signing request (CSR) files, import certificates, update working keys, and restart services. Common users can only view web server certificate information.

Viewing Web Server Certificate Information

Click on the Kunpeng Hyper Tuner page and choose Web Server Certificates.

The Web Server Certificates page is displayed, as shown in Figure 1. For details about the parameters, see Table 1.
Figure 1 Web Server Certificates page
Table 1 Parameter description

Parameter

Description

Certificate Name

Name of the certificate.

Certificate Expiration Time

Time when the certificate will expire.

Status

Certificate status. The values are as follows:

  • Valid: The remaining validity period is greater than the certificate expiration alarm threshold.
  • About to expire: The remaining validity period is less than or equal to the certificate expiration alarm threshold.
  • Expired: The certificate has expired.
NOTE:
  • The tool automatically checks and updates the certificate status at 02:00 every day.
  • By default, the expiration alarm threshold of web server certificates is 90 days. The administrator can click and choose System Settings to change the value of Web Service Certificate Expiration Alarm Threshold (days). The value ranges from 7 to 180 days.

Customizing and Importing a Server Certificate

The SSL certificate sets up an SSL security channel over HTTPS between the web browser on the client and the web server to transmit encrypted data between the client and server and prevent data disclosure. For security purposes, you are advised to replace the original certificate and keys with your customized certificate and public and private key pair, and promptly update the certificate.

  1. On the Web Server Certificates page, click Generate CSR File.

    The Generate CSR File dialog box is displayed, as shown in Figure 2. For details about the parameters, see Table 2.

    Figure 2 Generating a CSR File
    Table 2 Parameter description

    Parameter

    Description

    Country

    The user's country code.

    This parameter is mandatory. The value contains only two letters, for example, CN.

    Province

    Province of the user.

    The value can contain a maximum of 128 characters, including letters, digits, hyphens (-), underscores (_), periods (.), and spaces.

    City

    City of the user.

    The value can contain a maximum of 128 characters, including letters, digits, hyphens (-), underscores (_), periods (.), and spaces.

    Company

    Company of the user.

    The value can contain a maximum of 64 characters, including letters, digits, hyphens (-), underscores (_), periods (.), and spaces.

    Department

    Department of the user.

    The value can contain a maximum of 64 characters, including letters, digits, hyphens (-), underscores (_), periods (.), and spaces.

    Common Name

    Name of the user.

    This parameter is mandatory. The value can contain a maximum of 64 characters, including letters, digits, hyphens (-), underscores (_), periods (.), and spaces.

  2. Click OK.
  3. Send the exported CSR file to the SSL certificate issuer to apply for an SSL certificate.

    After obtaining the official certificate file (*.crt), save it to the local PC.

  4. Click Import Web Server Certificate.

    The Import Web Server Certificate page is displayed.

  5. Click Import and select the certificate file.

    • The certificate file to be imported must be in *.crt, *.cer, or *.pem format and cannot exceed 1 MB.
    • The CSR file generated in 1 correlates with the server certificate applied from the CA organization. Do not generate a new CSR file before importing the server certificate. Otherwise, the original CSR file is overwritten by the new CSR file and cannot be recovered. You have to use the new CSR file to apply for a new server certificate from the CA organization.

  6. Click OK.
  7. (Optional) Choose More > Restart Server to activate the certificate. If you do not restart the server immediately, the certificate will automatically take effect after the service restarts.

Updating the Working Key

The working key is used to encrypt the password for starting the NGINX service, and the root key is used to encrypt the working key. For security purposes, you are advised to update the working key periodically.

  1. Click in the upper right corner and choose Web Server Certificates.
  2. Choose More > Update Working Key.
  3. (Optional) Choose More > Restart Server to restart the NGINX service for the working key to take effect. If you do not restart the server immediately, the working key will automatically take effect after the service restarts.
