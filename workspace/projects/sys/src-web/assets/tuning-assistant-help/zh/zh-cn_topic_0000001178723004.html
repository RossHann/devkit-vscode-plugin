运行日志管理

前提条件

已成功登录系统性能分析。

只有管理员用户（tunadmin）可以查看和下载运行日志。

操作步骤

  1. 在页面右上角菜单选项中单击“日志”。

    打开“操作日志”界面。

  2. 单击“运行日志”页签。

    打开“运行日志”界面，如图1所示。

    图1 运行日志

  3. 单击指定运行日志“操作”列的“下载”。

    弹出操作确认对话框。

  4. 单击“确认”。

    下载运行日志时每100MB打包成一个压缩包。

    下载的运行日志的格式如下：

    时间|进程ID|线程ID|模块文件名:代码行数|函数名|级别|日志详情

    部分字段信息可能没有，请以实际情况为准。

    例如：

    2020-08-18 14:49:44,433|7806:281473041977840|taskqueue.py:165|start_task_in_task_queue|WARNING|taskqueue is empty

    日志级别用来表示日志信息的重要程度，工具运行日志级别分为DEBUG、INFO、WARNING、ERROR和CRITICAL，默认为WARNING，工具只会记录配置的级别及以上级别的日志。

    • DEBUG：调试级别，记录调试信息，便于开发人员或维护人员定位问题。
    • INFO：信息级别，记录服务正常运行的关键信息。
    • WARNING：警告级别，记录系统和预期的状态不一致的事件，但这些事件不影响整个系统的运行。
    • ERROR：一般错误级别，记录错误事件，但应用可能还能继续运行。
    • CRITICAL：严重错误级别，记录可能会导致系统崩溃的信息。
    • 管理员用户可在鲲鹏性能分析工具或系统性能分析首页中的菜单选项中选择“系统配置”，手动配置“用户管理运行日志级别”。
    • 管理员用户可在系统性能分析首页中的菜单选项中选择“系统配置”，手动配置“运行日志级别”。
