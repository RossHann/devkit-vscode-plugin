存储IO诊断

前提条件

已创建网络IO诊断任务，并且成功完成分析。

操作步骤

  1. 在“工程管理”区域，依次单击指定工程和分析任务名称前的。

    展开节点列表。

  2. 单击节点名称查看分析结果。

    单击节点名称默认打开“压测结果”页签，如图1所示，参数说明如表1所示。

    压测进程性能

    %user

    显示压测进程在用户态运行时所占用CPU总时间的百分比。

    %system

    显示压测进程在内核态运行时所占用CPU总时间的百分比。该指标没有包含服务硬件和软件中断所花费的时间。

    context switches

    显示压测进程切换次数。

    major page faults

    显示压测进程主缺页次数，当虚拟内存地址映射成物理内存地址时，相应的页在交换内存中，这样的缺页为主缺页（Major Page Faults），一般在内存使用紧张时产生，需要从硬盘中加载页。

    minor page faults

    显示压测进程次缺页次数，即虚拟内存地址映射成物理内存地址产生的缺页次数，不需要从硬盘中加载页。

    存储设备性能

    设备名称

    显示压测的设备名称。

    I/O number

    显示压测进程总共执行的IO次数。

    merge number

    显示压测进程总共发生的IO合并数。

    tick number

    显示磁盘繁忙的总滴答数。

    time spent in queue

    显示在队列中耗费的时间。

    util（%）

    显示磁盘利用率。
