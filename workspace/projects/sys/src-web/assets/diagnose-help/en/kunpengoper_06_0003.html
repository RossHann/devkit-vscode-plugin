Viewing Logs

  1. Use an SSH tool to remotely log in to the Linux CLI.
  2. Run the following command to go to the log file directory. Table 1 describes the log file names and paths.

    cd Log storage path

    Table 1 Log file description

    Log Type

    Log Name

    Path

    Description

    Installation log

    install.log

    Directory where the installation command is run (directory generated after the tool installation package is decompressed), for example, /home/Hyper_tuner.

    Records the installation of each component.

    Run log

    collection.log

    /Tool installation directory/hyper_tuner/sys-perf/logs/

    Records the run logs of analysis tasks.

    gunicorn_access.log
    • /Tool installation directory/hyper_tuner/sys-perf/logs
    • /Tool installation directory/hyper_tuner/user_management/logs

    Records the IP address and port for accessing gunicorn.

    gunicorn_error.log

    Records the gunicorn process ID and running error information.

    gunicorn.log

    Records gunicorn logs.

    c_analysis.log

    /Tool installation directory/hyper_tuner/sys-perf/logs/backend

    Records API logs of hotspot function task analysis.

    certificate_manager.log

    Contains machine-machine interface certificate reading records.

    certificate_manager_update_certificate.log

    Records certificate replacement operations.

    check_mem_access_task_param.log

    Records verification results of the input parameters for memory access analysis.

    check_system_lock_task_param.log

    Records verification results of the input parameters for lock and wait analysis.

    check_task_param.log

    Records the logs of check task parameters.

    common_analysis.log

    Records common analysis logs, including obtaining hotspot functions and function call stacks.

    config.log

    Records system configuration logs.

    data_sampling_control.log

    Records logs about checking drive space, checking logs, and starting and stopping tasks.

    data_sampling.log

    Records the collection process information.

    falsesharing_task.log

    Records the check of false sharing analysis input parameters.

    install_agent.log

    Records node installation information.

    java_analysis.log

    mem_access_analysis_control.log

    memaccessanalysis.log

    Records the Django logs of memory access analysis tasks.

    microarchitecture.log

    Records the data collection logs of microarchitecture analysis.

    Records the query of microarchitecture analysis.

    models.log

    Records task creation logs.

    node_manager.log

    Records logs of the node management module.

    operationlog_manager.log

    Records user operation logs.

    permission.log

    Records user management rights.

    polling_task.log

    Records the logs generated during the execution of schedule tasks.

    process_thread_check_task_param.log

    project_manager.log

    resource_analysis.log

    runlog_manager.log

    Records run logs.

    schedule_task.log

    system_config_control.log

    systemperformance.log

    task_manager.log

    Records logs of task management, including task deletion and editing.

    taskqueue.log

    user_manager.log

    work_key_manager.log

    user_manager.log

    Records user login management logs.

    work_key_manager.log
