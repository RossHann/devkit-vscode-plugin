Agent Service Certificate Management

The agent service certificate is used for communication between the Kunpeng Hyper Tuner server and agent.

Prerequisites

You have logged in to System Diagnosis.

The administrator (tunadmin) can generate certificates, replace certificates, and change working keys. Common users can only view Agent service certificate information.

Viewing Agent Service Certificate Information

  1. Click on the System Diagnosis page and choose Agent Service Certificates.
  2. View certificate details on the Agent Service Certificates page, as shown in Figure 1. For details about the parameters, see Table 1.

    Figure 1 Agent Service Certificates page
    Table 1 Parameters on the Agent Service Certificates page

    Parameter

    Description

    Node IP Address

    IP address of the node.

    Node Alias

    Alias of the node.

    Certificate Name

    Name of the certificate.

    Certificate Expiration Time

    Time when the certificate will expire.

    Status

    Certificate status. The values are as follows:

    • Valid: The remaining validity period is greater than the certificate expiration alarm threshold.
    • About to expire: The remaining validity period is less than or equal to the certificate expiration alarm threshold.
    • Expired: The certificate has expired.
    NOTE:
    • The tool automatically checks and updates the certificate status at 02:00 every day.
    • By default, the expiration alarm threshold of agent service certificates is 90 days. You can manually configure the threshold for the Agent service certificate expiration alarm. The value ranges from 7 to 180 days.

Creating and Replacing a Certificate

The SSL certificate sets up an SSL security channel over HTTPS between the Hyper Tuner server and an agent to transmit encrypted data between them and prevent data disclosure. For security purposes, you are advised to update the certificates periodically.

  • The certificate cannot be replaced during the execution of an analysis task.
  • Replace the certificate on the day when the certificate is generated.
  1. Click in the upper right corner and choose Agent Service Certificates.
  2. On the Agent Service Certificates page, click Create Certificate.
  3. Click Update Certificate in the Operation column of the node. A message is displayed, indicating that the certificate has been replaced.

Updating the Working Key

The working key is used to encrypt the private key files of the server and agent. For security purposes, you are advised to update the working key periodically.

The working key cannot be replaced during the execution of an analysis task.

  1. Click in the upper right corner and choose Agent Service Certificates.
  2. Click Update Working Key in the Operation column of the node. A message is displayed, indicating that the working key has been updated.
