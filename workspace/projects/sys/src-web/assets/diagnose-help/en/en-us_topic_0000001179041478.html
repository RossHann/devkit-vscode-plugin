Task Management

Prerequisites

  • A project has been created, and the project does not contain a node in Offline state.
  • Users can create analysis tasks only under their own projects.

Procedure

  1. Click next to the target project in the project list.

    The New Analysis Task page is displayed.

  2. Set task parameters.

    • You can click in the middle of the page to hide the banner on the home page.
    • You can click Import Template next to the Task Name text box to select and import a saved task template.
    Table 1 Parameters for adding a task

    Parameter

    Description

    Task Name

    Name of the task. The task name must meet the following requirements:

    • Contain only letters, digits, and special characters (including @#$ %^&*()[]<>._-!-+ and spaces).
    • Contain 1 to 32 characters.

    Diagnosis Object

    Select the object to be diagnosed.

    Selection options: Memory, Network I/O, Storage I/O

    Analysis Object

    This is not configurable. The default value is Application.

    This parameter is displayed when Diagnosis Object is set to Memory.

    Mode

    Collects data of a specified application or process for analysis.

    Mode for analysis object. The options are:
    • Launch Application: starts the application when the sampling begins. The sampling duration is determined by the execution time of the application. This option is recommended for applications with short execution time. The scenario analysis covers the application and its subprocesses.
    • Attach to Process: associates the existing PIDs on the server during data collection. You need to configure the sampling duration parameter (applicable to scenarios where the application runs for a long time). The scenario analysis covers only the PIDs in the parameter, not the sub-processes of the PIDs.

    Application Path

    Absolute path of the target application to be analyzed on the server. For example, to analyze the loop_test application stored in the /home/test directory, enter /home/test/loop_test.

    This parameter is displayed when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    NOTE:
    • By default, applications in the /opt/ or /home/ directory are analyzed. The administrator can click in the upper right corner on the home page of the System Diagnosis and choose System Settings, and enter application paths in the Application Path test box. Use semicolons (;) to separate multiple application paths. Only administrators can modify this parameter. Common users can only view this parameter.
    • The OS running user (malluma) of the Diagnosis and Debugging must have the execute permission on the application paths and the read and execute permissions on the applications to be analyzed.

    (Optional) Application Parameters

    Set application parameters based on the actual application scenario.

    This parameter is displayed when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    Process

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Attach to Process.

    Enter the name of the process to be analyzed.

    (Optional) PID

    This parameter is available when Memory is selected as the diagnosis object and Attach to Process is selected as the mode.

    Enter the PID of the process to be analyzed. You can enter multiple PIDs.

    (Optional) Application User

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application. By default, this parameter is disabled.

    • Disabled: The application user is the default system user launcher.
    • Enabled: The application user is the configured user.

    Analysis Type

    This parameter is not configurable. The default value is Memory Diagnosis Analysis.

    This parameter is available when Diagnosis Object is set to Memory.

    Diagnosis Content

    Diagnosis analysis type. This parameter is mandatory when Diagnosis Object is set to Memory.

    Possible values are:

    • Memory Leak
    • Memory Usage
    • OOM
    • Memory Overwriting Diagnosis
    NOTE:
    • Memory Overwriting Diagnosis cannot be selected together with Memory Leak, Memory Usage, and OOM. If Memory Overwriting Diagnosis is not selected, Memory Leak is mandatory, and Memory Usage and OOM are optional.
    • If Memory Overwriting Diagnosis is selected, you will be prompted to use GCC 4.9 or later and at least the -fsanitize=address option to perform compilation again.
    • If you select Memory Usage, ensure that the glibc version is later than 2.33. If the glibc version is earlier than 2.33, the statistics of the allocator may be inaccurate due to integer overflow.
    • The OOM diagnosis task and the storage I/O performance analysis task of the performance tool cannot be created at the same time.
    • When the Attach to Process mode is selected, the Memory Overwriting Diagnosis option is unavailable.

    (Optional) Binary/Symbol File Path

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    Absolute path of the binary or symbol file on the server.

    This parameter is not displayed when Diagnosis Content is set to Abnormal Memory Access.

    NOTE:
    • You can use this parameter to import the symbol table of an application when you need to view the performance data after source code and assembly instructions are mapped but the corresponding application has no symbol table information.
    • If the binary file of the application contains debug information, you only need to save the binary file in the directory.
    • If the binary file of the application does not contain debug information, save the binary file and the corresponding debug information file in the directory.

    (Optional) C/C++ Source File Path

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    Absolute path of the C/C++ source file on the server

    NOTICE:

    The source code of the application must comply with the general programming specifications. Otherwise, the source code of hot functions in the analysis result may not be displayed properly.

    NOTE:

    You can use this parameter to import the source code of an application to view the performance data after the source code and assembly instructions are mapped.

    Diagnosis Start Time (s)

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    Diagnosis start time. The sampling starts after the specified time. This parameter is used to ignore the program startup analysis, warm up the sampling program, and eliminate sampling delay caused by factors such as environment detection.

    This parameter is not displayed when Diagnosis Content is set to Memory Overwriting Diagnosis.

    Diagnosis Duration (s)

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    The period of time that the diagnosis lasts. By default, the diagonsis process continues until the program ends.

    This parameter is not displayed when Diagnosis Content is set to Memory Overwriting Diagnosis.

    Sampling Interval (ms)

    This parameter is available only when Diagnosis Object is set to Memory, Mode is set to Launch Application, and Diagnosis Content is set to Memory Usage.

    Sampling interval. The value ranges from 1 to 60000, and the default value is 1000. The smaller the interval, the greater the impact on the analysis process performance.

    Stop Analysis After Exception Occurs

    This parameter is available only when Diagnosis Object is set to Memory, Mode is set to Launch Application, and Diagnosis Content is set to Memory Overwriting Diagnosis.

    This function, which is enabled by default, stops the process when an exception occurs.

    Collect Call Stack

    This parameter is available when Diagnosis Object is set to Memory and Mode is set to Launch Application.

    This function is enabled by default.

    This parameter is not displayed when Diagnosis Content is set to Memory Overwriting Diagnosis.

    Size of the Collected File (MiB)

    Size of the collected file. The default value is 100 MiB. The value ranges from 1 to 100 MiB.

    You can set the size of the file to be collected to prevent long analysis time caused by large file size.

    Scheduled Startup

    Whether to create a scheduled analysis task. This parameter is disabled by default.

    1. Set Scheduled Startup to .

      The parameters for setting a scheduled task are displayed.

    2. Select the sampling mode for the scheduled task. The values are as follows:
      • Periodical: The task runs once a day in the sampling period.

        Set Sampling Time and Sampling Date for the periodic sampling task.

      • One-Time: The task runs only one time.

        Set Sampling Date and Time for the single sampling task.

    NOTE:
    • A maximum of 20 scheduled tasks can be created by all users.
    • The created scheduled task will be displayed in the scheduled task list. You can click in the upper right corner and choose Scheduled Tasks to view, edit, or delete (batch deletion supported) the scheduled tasks.

    Start Now

    Whether to automatically execute a task after it is created. This parameter is selected by default.

    NOTE:

    This parameter is not displayed if Scheduled Startup is enabled.

    Network I/O Diagnosis

    Diagnosis Scenario

    This parameter is mandatory when Diagnosis Object is set to Network I/O.

    Network I/O diagnosis scenarios. The options are:

    • Network dialing test
    • Packet loss diagnosis
    • Packet capture
    • Network load
    NOTE:

    Network dialing test or packet loss diagnosis must be selected. Packet capture and network load can be selected together.

    Dialing Test Scenario

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network dialing test.

    Select the dialing test scenario to be diagnosed. The options are:

    • Connectivity dialing test
    • TCP dialing test
    • UDP dial test

    IP Protocol Type

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network dialing test.

    Select the IP protocol type to be diagnosed. The options are:

    • IPv4
    • IPv6

    Node Information

    This parameter is mandatory when Diagnosis Object is set to Network I/O, Diagnosis Scenario is set to Network Dialing Test, and Dialing Test Scenario is set to Connectivity Dialing Test.

    • Task Node: Enter the IP address of the server where the task is executed.
    • Source IP Address (for IPv4): Enter the client IP address.
    • Target Server IP Address: Enter the IP address of the target server.
    • Source Network Port: This is required when an IPv6 address is used.
    NOTE:

    You can click to add a node or click to delete a node.

    Networking parameters

    This parameter is mandatory when Diagnosis Object is set to Network I/O, Diagnosis Scenario is set to Network Dialing Test, and Dialing Test Scenario is set to TCP Dialing Test or UDP Dialing Test.

    • Server Task Node
    • Server IP Address
    • Server Port
    • Client Task Node
    • Client IP Address
    • Client Port
    • Client Network Port (for IPv6)

    Report Interval (ms)

    This parameter is mandatory when Diagnosis Object is set to Network I/O, Diagnosis Scenario is set to Network Dialing Test, and Dialing Test Scenario is set to TCP Dialing Test or UDP Dialing Test.

    Interval for sending dialing test reports. The value ranges from 100 ms to 10000 ms. The default value is 1000 ms.

    Dialing Test Limit

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    • Dialing test duration (s): The default value is 10. The value ranges from 1s to 60.
    • Total length of dialing test packets: The value ranges from 1 KB to 1048576 KB.
    • Number of dialing test packets: The value ranges from 1 to 10000.

    (Optional) Dialing Test Packet Length (byte)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network dialing test.

    Length of a dialing test packet. The value ranges from 16 to 65507 bytes. The default value is 56 bytes.

    (Optional) Dialing Test Interval (ms)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network dialing test.

    Configure the dialing test interval. The value ranges from 10 ms to 10000 ms. The default value is 1000 ms.

    (Optional) Dialing Test Duration (s)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network dialing test.

    Configure the dialing test duration. The value ranges from 1s to 60s. The default value is 10s.

    (Optional) Fragmentation Policy

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to Connectivity.

    Options: want, do, and dont.

    NOTE:

    do: distribution is not allowed, even if the packet is discarded; want: packet is fragmented when it is too large; dont: DF flag is not set.

    (Optional) TTL

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to Connectivity.

    Configure the packet lifetime. The value ranges from 1 to 255. The default value is 30.

    Sampling time (s)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network load.

    Duration of load monitoring. The value ranges from 2 to 300, in seconds. The default value is 10.

    Collection Interval (s)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Network load.

    Indicates the collection interval of load monitoring. The value ranges from 1 to 10, in seconds. The default value is 10.

    Server CPU Affinity

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Configure the server CPU affinity and enter the CPU core ID.

    Client CPU Affinity

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Configure the client CPU affinity and enter the CPU core ID.

    Dialing test bandwidth

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Configure the bandwidth for the dialing test. The unit is Mbit/s and Kbit/s.

    Dialing Test Packet Length (byte)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Total length of the dialing test packet, for example, 123k/123M.

    Dialing Test Packets

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Configure the number of dialing test packets to be sent.

    Concurrent Connections

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Specifies the number of concurrent connections during a dialing test. The default value is 1.

    Socket buffer (byte)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Size of the socket buffer.

    MSS Length (Byte)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test.

    Set the maximum TCP fragment length. The value ranges from 88 to 1460 bytes, and the default value is 1460 bytes.

    Zero copy

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Dialing Test Scenario is set to TCP dialing test or UDP dialing test.

    Configure whether to support zero-copy data transmission.

    (Optional) Filter Criteria

    This parameter is optional when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet loss diagnosis.

    Configure the IP addresses and network adapter names to be filtered out.

    Collection Duration (s)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet loss diagnosis.

    Sampling duration for packet loss diagnosis. The value ranges from 1s to 300s, and the default value is 10s.

    Sampling Frequency (ms)

    (Optional)

    This parameter is mandatory when Diagnosis Object is set to Network I/O, Diagnosis Function is set to Packet Loss Diagnosis, and Sample Kernel Packet Loss Call Stack is enabled.

    Sampling frequency for packet loss diagnosis. The value ranges from 1 ms to 1000 ms, and the default value is 1 ms.

    Size of the Collected File (MiB)

    (Optional)

    This parameter is optional when Diagnosis Object is set to Network I/O, Diagnosis Function is set to Packet Loss Diagnosis, and Sample Kernel Packet Loss Call Stack is enabled.

    Size of the file to be sampled for packet loss diagnosis. The value ranges from 1 Mib to 1024 MiB, and the default value is 1024 MiB.

    Packet Capture Network Port

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    Configure the NIC for capturing network packets.

    Filter Conditions

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    • IP Protocol Type: specifies the IP protocol type for packet capture. This is optional. You can select more than one protocol.
    • IP1
    • Port 1
    • IP2
    • Port 2
    • Transfer Direction: specifies the transfer direction of the captured packets. The value can be Send, Receive, or Both.

    Packet Capture Duration (s)

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    Duration for capturing network packets. The value ranges from 1s to 300s, and the default value is 10s.

    Number of Captured Packets

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    Number of packets to be captured. The value ranges from 1 to 10000. The default value is 1000.

    File size

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    Size of the packet capture file. The default value is 100. The value ranges from 10 to 1024.

    Files

    This parameter is mandatory when Diagnosis Object is set to Network I/O and Diagnosis Scenario is set to Packet capture.

    Number of files for packet capture. The default value is 1. The value ranges from 1 to 10.

    Storage I/O Diagnosis

    Diagnosis Scenario

    This parameter is mandatory when Diagnosis Object is set to Storage I/O.

    Configure the diagnosis type. Storage I/O is selected by default.

    Pressure Test Object

    This parameter is mandatory when Diagnosis Object is set to Storage I/O.

    Select a pressure test object.

    • Node Name

      Select a node.

    • Node IP Address

      IP address of the selected node.

    • Logical Disk Name/File Name

      Name of the logical drive or file to be tested.

    NOTE:

    The pressure test on a logical drive will damage existing data on the drive.

    Key Metrics

    This parameter is mandatory when Diagnosis Object is set to Storage I/O.

    Pressure test parameters. Throughput, IOPS, and Latency are selected by default.

    • Request Block Size (KiB)
    • Read/Write Mode
    • Read/Write Ratio
    • I/O Queue Depth
    • I/O Engine
    • Concurrency
    • Direct I/O
    • I/O Size (MiB)
    • Test Duration
    • Related Metrics
    • Operation

      You can click Edit or Delete to perform corresponding operations on a model.

      NOTE:

      By default, there are 12 test models. You can edit or add models as required. A maximum of five models can be selected.

    Periodical Statistics

    This parameter is required when Diagnosis Object is set to Storage I/O.

    You can enable or disable this function. If it is disabled, data is collected at an interval of 500 ms.

    Statistical Period (ms)

    This parameter is required when Diagnosis Object is set to Storage I/O.

    Interval at which statistics are collected.

    Size of the Collected File (MiB)

    This parameter is required when Diagnosis Object is set to Storage I/O.

    Size of the file to be collected. The default value is 10 MiB. The value ranges from 1 MiB to 100 MiB.

  3. Set task parameters and click OK. For details about task parameters, see Table 1 and the related analysis task sections.

    You can click the icons next to the task name to perform the following operations:

    • : starts an analysis task. If Start Now is deselected when an analysis task is created, manually start the task.
    • : stops an analysis task. After an analysis task is stopped, the collected information will be deleted.
    • : restarts an analysis task. You can modify the task parameters and restart the analysis task.
    • : modifies an analysis task. Only the analysis tasks that are not started can be modified.
    • : deletes an analysis task. After a task is deleted, all data of this task will be deleted. Exercise caution when performing this operation.
    • The administrator (tunadmin) can view, restart, and delete analysis tasks created by all users.

      Common users can view, restart, and delete their own analysis tasks and view the analysis tasks created by other users.

    • You can click Save Template and enter the template name to save the configuration information of the current task as a template.

      The created template will be displayed in the task template list. You can click in the upper right corner and choose Task Templates to view and delete (batch deletion supported) task templates.

      A maximum of 10 templates can be saved for each analysis task type.

      All users can only view, import, and delete their own templates.

    After an analysis task is created, you can move the cursor onto the task name to view the detailed task information. Projects and analysis tasks can be imported and exported, as shown in Figure 1.
    • You can also click in the upper right corner of the page and choose Import/Export Task to view task records. You can view and delete imported tasks, and download and delete exported tasks. If an import or export task fails, you can click Retry to execute the task again.
    • The administrator can export projects and tasks created by all users. Common users can export only projects and tasks created by themselves.
    Figure 1 Icons for importing and exporting a project or analysis task

The specified file must have at least the read permission of the malluma user, and the directory where the file is located must have the access and write permissions of the malluma user.
