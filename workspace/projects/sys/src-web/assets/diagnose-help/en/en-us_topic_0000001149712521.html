Memory Overwriting Diagnosis

Prerequisites

A memory overwriting diagnosis task has been created and the analysis has been completed.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    The Memory Overwriting Diagnosis tab page is displayed by default, as shown in Figure 1.
    Figure 1 Memory overwriting diagnosis
    • In the source code box, the line that contains the code with abnormal memory access is highlighted.
    • You can use to switch the code lines that have abnormal memory access.

  3. Click to switch the view. See Figure 2.

    Figure 2 List view

    There are heap buffer overflow and global buffer overflow modules based on the exception type. The access type, unexpected access points, and auxiliary information are displayed for each module.

  4. Click View Details to view details about the exception.
  5. Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

    If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

  6. Click the Task Logs tab page to view the detailed log information.
