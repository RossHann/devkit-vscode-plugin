User Management

Prerequisites

You have logged in to Kunpeng Hyper Tuner or System Diagnosis.

  • Only the administrator tunadmin can create, modify, and delete users.
  • The user role specifies the operations that can be performed by the user:
    • tunadmin: default WebUI login user who has the administrator rights. This user can view, modify, and delete projects created by all users, and can view, restart, and delete analysis tasks created by all users.
    • User: users of this role can create projects and build analysis tasks. Create projects and analysis tasks, and view, modify, and delete their own projects, and view, restart, and delete their own tasks. View the projects and analysis tasks created by the user tunadmin or other users.

Procedure

  1. Click in the upper right corner and choose User Management.

    The User Management page is displayed.

  2. Click New.

    The New User dialog box is displayed, as shown in Figure 1. For details about the parameters, see Table 1.

    Figure 1 Creating a user
    Table 1 Parameters for creating a user

    Parameter

    Description

    Name

    Name of the user to be created. The user name must meet the following requirements:

    • Start with a letter.
    • Contain 6 to 32 characters.
    • Allow only letters, digits, hyphens (-), and underscores (_).

    Admin Password

    Password of the administrator.

    Password

    Password of the user to be created. The password must meet the following complexity requirements:

    • Contain 8 to 32 characters.
    • Contain at least two types of uppercase letters, lowercase letters, digits, and special characters (`~!@#$%^&*()-_=+\|[{}];:'",<.>/?).
    • Cannot be the same as the user name or the user name in reverse order.
    • Cannot be a password in the weak password dictionary.

    Confirm Password

    Password re-entered for confirmation.

    When a common user is created, the initial password is set. When logging in to the system upon the first time, the user must reset the password, and change this password periodically for security purposes.

  3. Configure the new user by referring to Table 1, and click OK.

    The created user is displayed in the user list on the User Management page.

    You can click the button in the Operation column of a specified user to perform the following operations:

    • Resetting the password: Enter the administrator password and new password to reset the user password.
    • Deletion: Enter the administrator password to delete a user. Deleting a user will delete all historical data of the user. Exercise caution when performing this operation.

      The administrator cannot delete a user who has projects. To delete such a user, delete its projects first.
