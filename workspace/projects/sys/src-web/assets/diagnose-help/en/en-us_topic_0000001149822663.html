Memory Usage

Prerequisites

The Memory Usage has been selected for the diagnosis content, a memory leak diagnosis task has been created, and the analysis has been completed.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    Click the node name and click the Memory Usage tab. See Figure 1.

    Figure 1 Memory usage
    Table 1 Memory Usage parameters

    Parameter

    Description

    Process

    Process name.

    You can select a process from the drop-down list or enter a process name in the search box to search for the process.

    Systems

    Displays the size of physical and virtual memories.

    Application

    Displays the requested memory size, number of request times, number of requested bytes, number of release times, number of released bytes, number of leaks, and number of leaked bytes.

    Allocator

    Displays the allocated memory, idle memory, used memory, number of Arenas, number of mmap areas, and mmap area size of the allocator.

    MAP Information of Process Memory

    Displays the size of the physical memory and virtual memory of each module.

  3. Click to switch the view. See Figure 2.

    Figure 2 List view

  4. Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

    If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

  5. Click the Task Logs tab page to view the detailed log information.
