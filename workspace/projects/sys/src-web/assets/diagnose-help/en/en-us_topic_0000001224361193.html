Run Log Management

Prerequisites

You have logged in to the System Diagnosis.

Only the administrator (tunadmin) can view and download run logs.

Procedure

  1. Click in the upper right corner and choose Logs.

    The Operation Logs page is displayed.

  2. Click the Run Logs tab.

    The Run Logs tab page is displayed, as shown in Figure 1.

    Figure 1 Run Logs tab page

  3. Locate the log file to be downloaded, and click Download in the Operation column.

    A confirmation dialog box is displayed.

  4. Click OK.

    A package will be created for run logs of every 100 MB.

    The downloaded run logs are in the following format:

    Time|Process ID|Thread ID|Module file name: Code line number|Function name|Level| Log details

    The actual log information displayed may vary.

    Example:

    2020-08-18 14:49:44,433|7806:281473041977840|taskqueue.py:165|start_task_in_task_queue|WARNING|taskqueue is empty

    The log level indicates the importance of the log information. The tool run logs are classified into five levels: DEBUG, INFO, WARNING, ERROR and CRITICAL. The default log level is WARNING. The tool records only the logs of the specified level and higher levels.

    • DEBUG: debugging information for fault locating.
    • INFO: key information about the normal running of the service.
    • WARNING: events about the system in unexpected status that does not affect the running of the system.
    • ERROR: errors that do not affect the application running.
    • CRITICAL: errors that may cause system breakdown.

    The administrator can click in the upper right corner on the home page of the Kunpeng Hyper Tuner or System Diagnosis and choose System Settings to change the value of Run Log Level.
