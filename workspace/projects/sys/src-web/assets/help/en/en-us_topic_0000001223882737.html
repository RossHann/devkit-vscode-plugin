Viewing Analysis Results

Prerequisites

A lock and wait analysis task is complete.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    • Click the node name. The Summary tab page is displayed by default, as shown in Figure 1. For details about the parameters, see Table 1.
      • The Summary tab page shows the number of call times of the lock and wait functions, associates detailed call information, and displays tuning suggestions. (The lock and wait functions that can be optimized are marked in red. You can move the cursor onto the function name to view detailed tuning suggestions.)
      • unknown indicates that the PC address of the function cannot be associated.
      Figure 1 Summary tab page
      Table 1 Parameters on the Summary tab page

      Parameter

      Description

      Task Name

      Displays the names of the lock and waiting tasks.

      Module Name

      Displays the name of the module corresponding to the lock and waiting task/invoking point.

      Function Name

      Displays the function name corresponding to the lock and waiting task/call point.

      Operation

      You can click View to view the function source code, assembly code, and code flow diagram.

      Call Times

      Number of times that the task is called.

      Timestamp

      Time point when the call stack is called.

      Call Site Source Code File Name

      Name of the source code file corresponding to the call site.

      Call Site Row Number

      Source code row number corresponding to the call site.

      Click View to view the function source code and assembly code analysis details. For details about the parameters, see Table 2.

      • The function details page displays the hotspot instructions (top instructions with the highest proportion of CPU Cycles events in a function) in the function, as well as high-level language files and line numbers corresponding to the hotspot instructions.
      • In the Assembly Code and Code Flow areas, you can perform flow control analysis on assembly code. You can see the "hot" levels of the assembly code blocks from the colors of the basic blocks and relationships between them.
      • In the Assembly Code area, you can click the up and down arrows to switch the code blocks.
      • If the Graphviz component is not installed in the operating system, the tool does not support the Code Flow function.
      Figure 2 Association between function source code and assembly instructions
      Table 2 Parameters on the function details page

      Parameter

      Description

      Hardware Event

      Type of the hardware event. In this version, only CPU Cycles is available.

      Total Events

      Total number of hardware events.

      File Name

      Name of the file where the current function is located.

      Source Code

      Line No.

      Line number of the source code.

      Source Code

      Source code.

      Count (%)

      Count: number of hardware events corresponding to the source code.

      %: percentage of the number of hardware events corresponding to the source code to the total number of events.

      Assembly Code

      Address

      Assembly instruction address.

      Line No.

      Line number of the source code corresponding to the assembly instruction.

      Source Code

      Assembly instructions executed.

      Count (%)

      Count: number of hardware events corresponding to the assembly instruction.

      %: percentage of the number of hardware events corresponding to the assembly instruction to the total number of events.
    • Click the Call Details tab page. The call details are displayed, as shown in Figure 3. For details about the parameters, see Table 3.
      • The Call Details page shows the time sequence data of the lock and wait functions called by each task.
      • unknown indicates that the PC address of the function cannot be associated.
      • You can click to select other tasks.
      Figure 3 Call Details tab page
      Table 3 Parameters on the Call Details tab page

      Parameter

      Description

      Task Time (s)

      Time when the call occurs.

      Module Name

      Displays the name of the module corresponding to the lock and waiting task/invoking point.

      Function Name

      Displays the function name corresponding to the lock and waiting task/call point.

      Source Code File Name

      Name of the source code file corresponding to the call site.

      Line No

      Source code row number corresponding to the call site.
    • Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

      If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

    • Click the Task Logs tab to view the logs about the sampling and analysis processes.
