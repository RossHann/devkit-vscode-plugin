Viewing Analysis Results

Prerequisites

A microarchitecture analysis task is complete.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    • Click the node name. The Summary tab page is displayed by default, as shown in Figure 1. For details about the parameters, see Table 1.
      • The Overview tab page provides metric data based on the top-down model. You can move the cursor onto a metric to view the metric description, exception description, possible causes, and solutions.
      • Click in the upper right corner to switch the view.
      Figure 1 Summary tab page
      Table 1 Parameters on the Summary tab page

      Parameter

      Description

      Clock Cycles

      Number of clock cycles in the sampling process.

      Instructions

      Number of instructions involved in the sampling process.

      IPC

      Number of instructions executed in a single clock cycle. IPC = Instructions/Clock Cycles

      Event Name

      Name of the event.

      Event Percentage

      Percentage of the current event to the total events.
    • Click Time Sequence Information to view the time sequence diagram of microarchitecture metrics, as shown in Figure 2.
      • The Time Sequence Information tab page displays metric information by time. You can specify different metrics based on time to identify features in different time segments. The tool provides the microarchitecture metric sequence diagram by process, thread, module, or CPU.
      • You can move the cursor onto a color stripe to view details and click the color stripe to view the child metrics.
      Figure 2 Time Sequence Information tab page
    • Click the Details tab to view detailed information about microarchitecture metrics, as shown in Figure 3. Table 2 lists the parameters and descriptions.
      • The Detail tab page is displayed when Sampling Mode is set to Detail.
      • The Details tab page provides micro metrics of processes, threads, modules, and functions by CPU usage.
      • unknown indicates that the program counter (PC) address of the function cannot be associated.
      • You can select a function view from the Function/Callstack drop-down list box.

        Function/Callstack: displays function information.

        Module/Function/Callstack: displays function information by module.

        Thread/Function/Callstack: displays function information by thread.

        Core/Thread/Function/Callstack: displays function information by core.

        Processes/Function/Thread/Callstack: displays function information by process.

        Processes/Thread/Module/Function/Callstack: displays function information by process.

        Processes/Module/Thread/Function/Callstack: displays function information by process.

        Processes/Module/Function/Thread/Callstack: displays function information by process.

        Function/Thread/Core/Callstack: displays function information by function. You can click the drop-down button on the left of a function name and move the cursor onto a thread ID to view detailed command line information.

      Figure 3 Details tab page
      Table 2 Parameters on the Details tab page

      Parameter

      Description

      Clock Cycles

      Number of clock cycles in the sampling process.

      Instructions

      Number of instructions involved in the sampling process.

      IPC

      Number of instructions executed in a single clock cycle. IPC = Instructions/Clock Cycles

      Module Path

      Absolute path of the module.

      Clock Cycle Proportion

      Percentage of the clock cycle used by the current call stack.

      Instruction Proportion

      Percentage of the instructions used by the current call stack.

      PID

      Process ID.

      TID

      Thread ID.
      Click a function name to view the function source code and assembly code analysis details, as shown in Figure 4. For details about the parameters, see Table 3.

      The function details page displays the hotspot instructions (top instructions with the highest proportion of CPU Cycles events in a function) in the function, as well as high-level language files and line numbers corresponding to the hotspot instructions.

      Figure 4 Association between function source code and assembly instructions
      Table 3 Parameters on the function details page

      Parameter

      Description

      Hardware Event

      Type of the hardware event. In this version, only CPU Cycles is available.

      Total Events

      Total number of hardware events.

      File Name

      Name of the file where the current function is located.

      Source Code

      Line No.

      Line number of the source code.

      Source Code

      Source code.

      Count (%)

      Count: number of hardware events corresponding to the source code.

      %: percentage of the number of hardware events corresponding to the source code to the total number of events.

      Assembly Code

      Address

      Assembly instruction address.

      Line No.

      Line number of the source code corresponding to the assembly instruction.

      Source Code

      Assembly instructions executed.

      Count (%)

      Count: number of hardware events corresponding to the assembly instruction.

      %: percentage of the number of hardware events corresponding to the assembly instruction to the total number of events.
    • Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

      If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

    • Click the Task Logs tab page to view the log information about the task.
