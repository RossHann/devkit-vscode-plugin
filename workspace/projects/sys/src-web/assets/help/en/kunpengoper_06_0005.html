Replacing the SSL Certificate of the Kunpeng Hyper Tuner

The Secure Sockets Layer (SSL) certificate helps establish an encrypted link (accessed using HTTPS) between a web server and a browser to ensure secure data transmission. In addition, you can use the server certificate to check whether the website you access is authentic and reliable. The Kunpeng Hyper Tuner supports the SSL certificate replacement function. You are advised to replace the certificate and public-private key pair with your own ones with timely updates.

  1. Register a domain name, apply for an SSL certificate, and name the SSL certificate files as follows:

    • Server certificate file: server.crt
    • Server certificate signing request file: server.csr
    • Private key file: server.key
    • Certificate chain file: ca_chain.crt

      The identity of a public key certificate must be authenticated by the certificate authority (CA) of the upper level. The process for authenticating a public key becomes an iterative process. As a result, a certificate chain is formed, which ends at the root certificate. The file recording the iterative process is referred to as a certificate chain file.

  2. Use an SSH remote login tool to upload the obtained certificate file to any directory on the server, for example, the /home directory.
  3. Use an SSH remote login tool to log in to the Linux CLI as the system user.

    This step must be performed as the root user. If you log in as a common system user, run the su command to switch to the root user and perform subsequent operations.

  4. Run the following command to go to the directory where the decompressed Kunpeng Hyper Tuner installation package is stored. (Replace /home with the actual directory.)

    cd /home/Hyper_tuner

  5. Run the following command to replace the SSL certificate:

    ./install_service_cert.sh [-D domain] [-s cert directory] [-r True/False] [-c True/False]

    Example:

    ./install_service_cert.sh -D *.huawei.com -s /home

    Conventions for the command line formats:

    • Boldface: The keywords that must keep unchanged in a command line are in boldface.
    • Italic: Command arguments that must be replaced by specific values are in italic.
    • [ ]: Items in square brackets [ ] are optional.
    Table 1 Parameter description

    Command

    Parameter

    Description

    -D/--domain

    domain

    (Optional) Domain name. You can enter multiple domain names and separate them with commas (,). This parameter is mandatory when you configure -s/--source to replace the certificate with your own certificate.

    -s/--source

    cert directory

    (Optional) Directory where certificates are uploaded. By default, the Huawei certificate is installed.

    -r/--replace

    True/False

    (Optional) Whether to replace the certificate. The default value is False. If a certificate has been installed, set this parameter to True.

    -c/--check-crl

    True/False

    (Optional) Whether to check the CA certificate revocation list (CRL). The default value is False.

    NOTE:

    To check the CA CRL, access the Internet and verify the CRL with the CA.

  6. Open the browser on the local PC, enter https://Domain name:Port number in the address box to log in to the Kunpeng Hyper Tuner WebUI.
