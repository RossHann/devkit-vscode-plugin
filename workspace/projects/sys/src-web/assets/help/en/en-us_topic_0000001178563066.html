Viewing Analysis Results

Prerequisites

An HPC analysis task is complete.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    • Click a node name. The Summary tab page is displayed by default. See Figure 1. Table 1 describes the parameters.
      Figure 1 Summary
      Table 1 Parameters on the Summary tab page

      Parameter

      Description

      Running Time

      Displays execution time of an application.

      Serial time

      Displays serial running time of an application.

      Parallel Time

      Displays how long an application is running in parallel.

      Imbalance Time

      Displays the imbalanced running time of an application.

      CPI

      Displays the ratio of CPU cycles/Retired instructions, which indicates the clock cycle consumed by each instruction.

      CPU Usage

      Displays the CPU usage, that is, the ratio of the CPU usage to the running OpenMP.

      OpenMP Team Usage

      Displays the usage of the OpenMP Team.

      Hotspots

      By default, Functions is displayed. You can also select Module, parallel-region, or barrier-to-barrier-segment.
    • Click the HPC Metrics tab page. The HPC metric analysis result is displayed, as shown in Figure 2 and Figure 3. For details about the parameters, see Table 2.

      You can click to switch between the view modes.

      Figure 2 HPC Metrics tab
      Figure 3 HPC indicator (PMU events)
      Table 2 HPC metric parameters

      Parameter

      Description

      Memory Bandwidth

      Average DRAM Bandwidth

      Displays the average DRAM bandwidth.

      Read Bandwidth

      Displays the average read bandwidth.

      Write Bandwidth

      Displays the average write bandwidth.

      Intra-Socket Bandwidth

      Displays the bandwidth of a socket.

      Cross-Socket Bandwidth

      Displays the cross-socket bandwidth.

      L3 By-Pass Rate

      Displays the L3 bypass rate.

      L3 Miss Rate

      Displays the L3 miss rate.

      L3 Usage

      Displays the L3 cluster usage.

      Command distribution (move the mouse to the question mark next to a parameter to view details)

      HPC Top-Down

      Event Name

      Name of the Top-Down event.

      Event Percentage

      Proportion of the Top-Down event.

      OpenMP runtime indicators

      Serial time (s)

      Displays the serial running time of OpenMP.

      Parallel time (s)

      Displays the parallel running time of OpenMP.

      Parallel region

      Displays the parallel area.

      Barrier-to-barrier segment

      Displays a special stand-alone section.

      Time used (s)

      Total execute time of OpenMP in seconds.

      Imbalance time (s)

      Displays the running time of OpenMP that are imbalanced.

      Imbalance rate (%)

      Displays the running rate of OpenMP that are imbalanced.

      Potential Gain(s)

      Displays the difference between the actual duration and the theoretical duration.

      CPU Utilization(%)

      Displays the CPU usage.

      Spin(%)

      Displays the CPU waiting rate for the OS and parallel processing.

      Overhead(%)

      Displays the CPU parallel refined ratio.

      Lock Contention(s)

      Lock contention time.

      Creation(s)

      Duration of parallel work.

      Scheduling(s)

      Scheduling time.

      Tasking(s)

      Time when the task is assigned.

      Reducation(s)

      Reduction operation time.

      Atomics(s)

      Atom operation time.

      MPI runtime metrics

      Rank

      The logical working unit is displayed.

      Elapsed Time (s)

      Running duration.

      Overall Wait Rate (%)

      Percentage of waiting time.

      Communication (%)

      Percentage of the communication waiting time.

      P2P Communication (%)

      Percentage of the point to point communication waiting time.

      Collective Communication (%)

      Percentage of the collective communication waiting time.

      Synchronization (%)

      Percentage of the synchronization waiting time.

      Single I/O (%)

      Percentage of the single I/O waiting time.

      I/O (%)

      Percentage of the I/O waiting time.

      Collective I/O (%)

      Percentage of the collective I/O waiting time.

      Data Size (bytes)

      Size of transmitted data.

      Function

      Invoked function.

      Call Count

      Number of invocations.

      Send data type

      Type of the sent data.

      Recv data type

      Type of the received data.

      mpi-comm

      Information about the MPI communication domain.

      Number of original PMU events

      Event

      Name of the PMU event.

      Count

      Number of PMU events.
    • Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

      If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

    • Click the Task Logs tab page to view the detailed log information.
