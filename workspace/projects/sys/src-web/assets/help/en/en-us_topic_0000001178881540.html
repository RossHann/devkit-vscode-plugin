Viewing Analysis Results

Prerequisites

A hotspot function analysis task is complete.

Procedure

  1. In the Project Management area, click before the target project and analysis task.

    The node list is displayed.

  2. Click the name of the target node to view the analysis result.

    • Click the node name. The Summary tab page is displayed by default, as shown in Figure 1. For details about the parameters, see Table 1.
      • The Summary page displays tuning suggestions, statistics information, platform information, and top 10 hotspot functions.
      • The Tuning Suggestions area is displayed only when the system detects metrics to be optimized. You can expand the area to view detailed suggestions and solutions.
      • If an optimized version of the Kunpeng computing platform is available for a hotspot function in the Top 10 Hotspots area, the function is marked with , and the tuning suggestions, download button, and URL copy button are displayed in a pop-up message.
      • unknown indicates that the PC address of the function cannot be associated.
      Figure 1 Summary tab page
      Table 1 Parameters on the Summary tab page

      Parameter

      Description

      Statistics

      Execution Time (s)

      Execution time of the program.

      NOTE:

      This parameter is displayed when Mode is set to Launch Application during task creation.

      Data Collection Time (s)

      Sampling duration, in seconds.

      NOTE:

      This parameter is displayed when Analysis Object is set to System or Mode is set to Attach to Process during task creation. The value of this parameter is the same as the value of Collection Duration (s) set when an analysis task is created.

      Clock Cycles

      Number of clock cycles in the sampling process.

      Instructions

      Number of instructions involved in the sampling process.

      IPC

      Number of instructions executed in a single clock cycle. IPC = Instructions/Clock Cycles

      Platform Info

      Operating System

      Version of the operating system in use.

      Host Name

      Name of the host.

      Top 10 Hotspots

      Function

      Function name.

      Module

      Module to which the function belongs.

      Clock Cycles

      Number of clock cycles required for function execution.

      Clock Cycle Proportion

      Percentage of the clock cycles executed by the function to all the clock cycles.

      Execution Time (s)

      Execution time of the function.
    • Click the Function tab page. The function analysis result is displayed, as shown in Figure 2. For details about the parameters, see Table 2.
      • On the Function tab page, the function execution information and call stack information are provided in different dimensions.
      • unknown indicates that the PC address of the function cannot be associated.
      • You can select a function view from the Function/Callstack drop-down list box.

        Function/Callstack: displays function information.

        Module/Function/Callstack: displays function information by module.

        Thread/Function/Callstack: displays function information by thread.

        Core/Function/Callstack: displays function information by CPU core. (This option is displayed when Analysis Object is set to System.)

        Function/Thread/Core/Callstack: displays function information by function. You can click the drop-down button on the left of a function name and move the cursor onto a thread ID to view detailed command line information.

      Click a function name to view the function source code and assembly code analysis details, as shown in Figure 3. For details about the parameters, see Table 3.

      • The function details page displays the hotspot instructions (top instructions with the highest proportion of CPU Cycles events in a function) in the function, as well as high-level language files and line numbers corresponding to the hotspot instructions.
      • In the Assembly Code and Code Flow areas, you can perform flow control analysis on assembly code. You can see the "hot" levels of the assembly code blocks from the colors of the basic blocks and relationships between them.
      • In the Assembly Code area, you can click the up and down arrows to switch the code blocks.
      • If the Graphviz component is not installed in the operating system, the tool does not support the Code Flow function.

    • Click the Hot Flame Graph tab page. The hot flame graph is displayed, as shown in Figure 4.
      • The flame graph is described as follows:

        The y axis indicates the call stack. Each layer has a function. A deeper stack indicates a higher flame. The top is the function being executed, and its parent functions are below it.

        The x axis indicates the number of samples. The width of a function on the x axis indicates the number of times that the function is sampled. A wider function indicates longer execution time of the function. Note that the x axis does not represent time. It is arranged alphabetically after all call stacks are combined.

      • Function call relationship and execution time can be displayed in the flame graph, helping to find the hotspot functions and their tuning paths. The hot flame graph indicates the CPU usage and is used to locate high CPU usage. Generally, warm colors are used.
      • When you move the cursor onto a function block, a floating box is displayed. You can click View Function Details, if displayed, to view the function details.
      Figure 4 Hot Flame Graph tab page
    • Click the Cold Flame Graph tab page. The cold flame graph is displayed, as shown in Figure 5.
      • The flame graph is described as follows:

        The y axis indicates the call stack. Each layer has a function. A deeper stack indicates a higher flame. The top is the function being executed, and its parent functions are below it.

        The x axis indicates the number of samples. The width of a function on the x axis indicates the number of times that the function is sampled. A wider function indicates longer execution time of the function. Note that the x axis does not represent time. It is arranged alphabetically after all call stacks are combined.

      • Function call relationship and execution time can be displayed in the flame graph, helping to find the hotspot functions and their tuning paths. The cold flame graph is used to locate low CPU usage. Generally, cold colors are used.
      • When you move the cursor onto a function block, a floating box is displayed. You can click View Function Details, if displayed, to view the function details.
      Figure 5 Cold Flame Graph tab page
    • Click the Task Information tab to view the detailed configuration and sampling information about the task on the current node.

      If the task fails to be executed, the failure cause is displayed on the Task Information tab page.

    • Click the Task Logs tab to view the logs about the sampling and analysis processes.
