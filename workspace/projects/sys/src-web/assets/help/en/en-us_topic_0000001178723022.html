Creating an HPC Analysis Task

Function

HPC analysis collects PMU events of the system and the key metrics of OpenMP and MPI applications to help users accurately obtain the serial and parallel time of the Parallel region and Barrier-to-Barrier, calibrated 2-layer micro-architecture metrics, instruction distribution, L3 usage, and memory bandwidth.

Prerequisites

An HPC project has been created, and the project does not contain a node in Offline state.

Procedure

  1. Click next to the target project in the project list.

    The New Analysis Task page is displayed.

  2. Set task parameters by referring to Task Management and Table 1.

    Create an HPC analysis task, as shown in Figure 1.
    Figure 1 Creating an HPC analysis task
    Table 1 Parameters for creating an HPC analysis task

    Parameter

    Description

    Task Name

    Name of the task. The task name must meet the following requirements:

    • Contain only letters, digits, and special characters (including @#$ %^&*()[]<>._-!-+ and spaces).
    • Contain 1 to 32 characters.

    Analysis Object

    Select Systems or Application.

    Mode

    Select Launch Application or Attach to Process.

    This parameter is mandatory when Analysis Object is set to Application.

    Application Path

    Enter the absolute path of the application to be analyzed. For example, to analyze the loop_test application stored in the /home/test directory, enter /home/test/loop_test.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Launch Application.

    NOTE:
    • By default, applications in the /opt/ or /home/ directory are analyzed. The administrator can click in the upper right corner on the home page of the System Profiler and choose System Settings, and enter application paths (separated by semicolons) in the Application Path test box. Only administrators can modify this parameter. Common users can only view this parameter.

      You are advised to set the application path to a path such as /home or /opt. Do not set the application path to a system directory such as /, /dev, /sys, or /boot. Otherwise, system exceptions may occur.

    • The OS running user (launcher) of the Kunpeng Hyper Tuner must have the read and execute permissions on the applications to be analyzed.
    • In the multi-node scenario, you can enable Configure Node Parameters to configure this parameter separately for each node.

    (Optional) Application Parameters

    Enter application parameters based on the actual scenario.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Launch Application.

    NOTE:

    You can enable Configure Node Parameters to configure this parameter separately for each node.

    Analysis Type

    Select HPC Analysis

    Sampling Time (s)

    Sampling duration, in seconds. The default value is 30. The value range is 1 to 300. If the value exceeds the range, a message is displayed, and a backfill is automatically performed after the focus is lost.

    (Optional) Sampling Type

    You can select one of the following sampling types:

    • Summary
    • Instruction distribution
    • HPC Top-Down

    (Optional) OpenMP

    Set the OpenMP environment variables. Separate variables with spaces. Example: OMP_NUM_THREADS=32 OMP_PROC_BIND=88. The default value is OMP_NUM_THREADS=32.

    This parameter is required when the analysis object is set to Application, the analysis mode is set to Launch Application, and the sampling type is set to OpenMP.

    Configure Node Parameter

    Set node parameters based on the site requirements. If you do not need to set parameters for each node separately, all nodes are executed based on the current parameter settings.

    This parameter is required when the analysis object is set to Application, the analysis mode is set to Launch Application, and the sampling type is set to OpenMP.

    Sampling Type

    Available sampling types. Options:

    • OpenMP
    • MPI

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Launch Application.

    MPI Command Directory

    Input the directory which stores the MPI commands.

    This parameter is displayed when Analysis Object is set to Application, Mode is set to Launch Application, and Sampling Type is set to MPI.

    (Optional) MPI Parameter

    Input MPI running parameters.

    This parameter is displayed when the analysis object is set to Application, the analysis mode is set to Launch Application, and the sampling type is set to MPI.

    Select the node where the mpirun process is running.

    Select the node where MPIRUN runs.

    This parameter is required when Analysis Object is set to Application, Mode is set to Launch Application, and Sampling Type is set to MPI.

    Configure Rank Quantity

    Click the button for configuring the number of ranks. In the dialog box that is displayed, configure the number of ranks of the node. The default value is 4 and the value range is 1-128.

    This parameter is required when the analysis object is set to Application, the analysis mode is set to Launch Application, and the sampling type is set to MPI.

    (Optional) Application User

    This parameter is unavailable by default. It is available only when Analysis Object is set to Application and Mode is set to Launch Application. When Sampling Type is set to MPI, this parameter is automatically enabled.

    • Disabled: The application user is the default system user launcher.
    • Enabled: The application user is the configured user.

    User Name

    Enter the name of the system user who starts the application.

    This parameter is displayed when Analysis Object is set to Application, Mode is set to Launch Application, and Application User is configured.

    Password

    Enter the password.

    This parameter is displayed when Analysis Object is set to Application, Mode is set to Launch Application, and Application User is configured.

    Name

    Enter the name of the process to be analyzed. Regular expressions are supported.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Attach to Process.

    PID

    Enter the PID of the process to be analyzed. Separate PIDs with commas (,).

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Attach to Process.

    Scheduled Startup

    Whether to create a scheduled analysis task. For details, see Task Management.

    Start Now

    Whether to automatically execute a task after it is created. This parameter is selected by default.

    NOTE:

    This parameter is not displayed if Scheduled Startup is enabled.

  3. Click OK.
