Task Management

Prerequisites

  • A project has been created, and the project does not contain a node in Offline state.
  • Users can create analysis tasks only under their own projects.

Procedure

  1. Click next to the target project in the project list.

    The New Analysis Task page is displayed.

  2. Set task parameters.

    • You can click in the middle of the page to hide the banner on the home page.
    • You can click Import Template next to the Task Name text box to select and import a saved task template.
    • This section describes only common task parameters. For details about how to set other task parameters, see the corresponding analysis task sections.
    • The L3C access bandwidth and hit rate are not displayed in some systems.
    • Some VMs (such as HUAWEI CLOUD ECS) do not support the vPMU. As a result, the performance analysis task for collecting hardware PMU events will fail on these VMs.
    Table 1 Parameters for adding a task

    Parameter

    Description

    Task Name

    Name of the task. The task name must meet the following requirements:

    • Contain only letters, digits, and special characters (including @#$ %^&*()[]<>._-!-+ and spaces).
    • Contain 1 to 32 characters.

    Analysis Object

    Object to be analyzed. The values are as follows:

    • System: collects performance data of the server, regardless of the types of applications running in the system. The sampling duration is determined by the parameter specified. This option is recommended for servers running multiple types of services and subprocesses.
    • Application: collects data of a specified application or process for analysis.
      After selecting Application, select a mode for the analysis object. The values are as follows:
      • Launch Application: starts the application when the sampling begins. The sampling duration is determined by the execution time of the application. This option is recommended for applications with short execution time. The scenario analysis covers the application and its subprocesses.
      • Attach to Process: associates the existing PIDs on the server during data collection. You need to configure the sampling duration parameter (applicable to scenarios where the application runs for a long time). The scenario analysis covers only the PIDs in the parameter, not the sub-processes of the PIDs.
      NOTE:

      The parameters to be configured vary with the analysis type. For details, see the corresponding analysis task sections.

    Analysis Type

    Type of the analysis task to be performed.

    The system analysis objects include:

    • General Analysis
      • Overall Analysis
      • Process/Thread Performance
      • Hotspot Function Analysis
    • System Component Analysis
      • Microarchitecture
      • Memory Access Analysis
      • I/O Performance
    • Specified Analysis
      • Resource Scheduling
      • Locks and Waits
      • HPC (only in HPC scenarios)

    The application analysis objects include:

    • General Analysis
      • Process/Thread Performance
      • Hotspot Function Analysis
    • System Component Analysis
      • Microarchitecture
      • Memory Access Analysis
      • I/O Performance
    • Specified Analysis
      • Resource Scheduling
      • Locks and Waits
      • HPC (only in HPC scenarios)
    NOTE:
    • The parameters to be configured vary with the analysis type. For details, see the corresponding analysis task sections.
    • The microarchitecture analysis task supports only the physical machine scenario.

    Configure Node Parameters
    Whether to set parameters of each node separately. This parameter is disabled by default, indicating that the task will be executed based on the current parameter configuration on all nodes.
    NOTE:

    This parameter is displayed when the project contains two or more nodes.

    After setting the mandatory parameters, which marked with an asterisk (*), you can enable this option () to set parameters for each node separately.

    1. Click Configure Parameters of a node in the Operation column.

      The page for setting parameters of the node is displayed.

    2. Configure parameters of the node as required and click OK.

      The Configuration Status of the node becomes Configured.

    3. Repeat 1 to 2 to set parameters of the other nodes.

    Scheduled Startup

    Whether to create a scheduled analysis task. This parameter is disabled by default.

    1. Set Scheduled Startup to .

      The parameters for setting a scheduled task are displayed.

    2. Select the sampling mode for the scheduled task. The values are as follows:
      • Periodical: The task runs once a day in the sampling period.

        Set Sampling Time and Sampling Date for the periodic sampling task.

      • One-Time: The task runs only one time.

        Set Sampling Date and Time for the single sampling task.

    NOTE:
    • A maximum of 20 scheduled tasks can be created by all users.
    • The created scheduled task will be displayed in the scheduled task list. You can click in the upper right corner and choose Scheduled Tasks to view, edit, or delete (batch deletion supported) the scheduled tasks.

    Start Now

    Whether to automatically execute a task after it is created. This parameter is selected by default.

    NOTE:

    This parameter is not displayed if Scheduled Startup is enabled.

  3. Set task parameters and click OK. For details about task parameters, see Table 1 and the related analysis task sections.

    You can click the icon next to the task name to perform the following operations:

    • : starts an analysis task. If Start Now is deselected when an analysis task is created, manually start the task.
    • : stops an analysis task. After an analysis task is stopped, the collected information will be deleted.
    • : restarts an analysis task. You can modify the task parameters and restart the analysis task.
    • : modifies an analysis task. Only the analysis tasks that are not started can be modified.
    • : deletes an analysis task. After a task is deleted, all data of this task will be deleted. Exercise caution when performing this operation.
    • The administrator (tunadmin) can view, restart, and delete analysis tasks created by all users.

      Common users can view, restart, and delete their own analysis tasks and view the analysis tasks created by other users.

    • You can click Save Template and enter the template name to save the configuration information of the current task as a template.

      The created template will be displayed in the task template list. You can click in the upper right corner and choose Task Templates to view and delete (batch deletion supported) task templates.

      A maximum of 10 templates can be saved for each analysis task type.

      All users can only view, import, and delete their own templates.

      For restarting an analysis task, the values of Task Name, Analysis Object, and Analysis Type are the same as those set during task creation and cannot be edited, and the Scheduled Startup option is unavailable.

    After an analysis task is created, you can move the cursor onto the task name to view the detailed task information. Projects and analysis tasks can be imported and exported. See Figure 1.
    • You can also click in the upper right corner of the page and choose Import/Export Task to view task records. You can view and delete imported tasks, and download and delete exported tasks. If an import or export task fails, you can click Retry to execute the task again.
    • The administrator can export the projects and tasks created by all users. Common users can export only projects and tasks created by themselves.
