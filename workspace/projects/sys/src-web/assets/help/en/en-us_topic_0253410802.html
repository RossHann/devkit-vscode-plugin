Agent Service Certificate Management

The agent service certificate is used for communication between the Kunpeng Hyper Tuner server and agent.

Prerequisites

You have logged in to the System Profiler WebUI.

The administrator (tunadmin) can generate certificates, replace certificates, and change working keys. Common users can only view Agent service certificate information.

Viewing Agent Service Certificate Information

  1. Click on the System Profiler page and choose Agent Service Certificates.
  2. View certificate details on the Agent Service Certificates page, as shown in Figure 1. For details about the parameters, see Table 1.

    Figure 1 Agent Service Certificates page
    Table 1 Parameters on the Agent Service Certificates page

    Parameter

    Description

    Node IP Address

    IP address of the node.

    Node Alias

    Alias of the node.

    Certificate Name

    Name of the certificate.

    Certificate Expiration Time

    Time when the certificate will expire.

    Status

    Certificate status. The values are as follows:

    • Valid: The remaining validity period is greater than the certificate expiration alarm threshold.
    • About to expire: The remaining validity period is less than or equal to the certificate expiration alarm threshold.
    • Expired: The certificate has expired.
    NOTE:
    • The tool automatically checks and updates the certificate status at 02:00 every day.
    • By default, the expiration alarm threshold of agent service certificates is 90 days. The administrator can click in the upper right corner on the home page of the System Profiler and choose System Settings to change the value of Agent Service Certificate Expiration Alarm Threshold (days). The value ranges from 7 to 180 days.

Creating and Replacing a Certificate

The SSL certificate sets up an SSL security channel over HTTPS between the Hyper Tuner server and an agent to transmit encrypted data between them and prevent data disclosure. For security purposes, you are advised to update the certificates periodically.

  • The certificate cannot be replaced during the execution of an analysis task.
  • Replace the certificate on the day when the certificate is generated.
  • The dialog box for replacing a certificate is displayed only when you replace a certificate on an Agent node.
  1. On the Agent Service Certificates page, click Create Certificate.
  2. Click Update Certificate in the Operation column of a node.

    The Update Certificate dialog box is displayed, as shown in Figure 2 (The node installed by the root user is used as an example.). For details about the parameters, see Table 2.
    Figure 2 Replacing a certificate
    Table 2 Parameters for updating a certificate

    Parameter

    Description

    Node IP Address

    IP address of the server where the node is installed.

    User Name

    User name of the installation node. The default value is root.

    Authentication Mode

    Authentication mode. The values are as follows:

    • Password authentication
    • Key authentication
    NOTE:

    If you select Key authentication, configure SSH authentication information on the server where the Kunpeng Hyper Tuner is installed. For details, see Common Operations > Configuring SSH Authentication Information on a Node Server.

    Password

    Password used when the node is added.

    NOTE:
    • This parameter is displayed when Authentication Mode is set to Password authentication.
    • If the node is installed by the root user, you need to enter the password of the root user. If the node is installed by a common user, you need to enter the passwords of both the common user and root user.

    Private Key File

    Absolute path of the SSH private key file for logging in to the node server.

    NOTE:

    This parameter is displayed when Authentication Mode is set to Key authentication.

    Passphrase

    Password of the SSH private key file for logging in to the node server. If the SSH private key password is not configured, you do not need to set this parameter.

    NOTE:

    This parameter is displayed when Authentication Mode is set to Key authentication.

  3. Click OK.

Updating the Working Key

The working key is used to encrypt the private key files of the server and agent. For security purposes, you are advised to update the working key periodically.

The working key cannot be replaced during the execution of an analysis task.

  1. Click in the upper right corner and choose Agent Service Certificates.
  2. Click Update WorkKey in the Operation column of a node.

    The Update Working Key dialog box is displayed, as shown in Figure 3. For details about the parameters, see Table 3.

    • Key authentication
