Creating an Overall Analysis Task

Function Description

The tool collects information about CPU, memory, storage I/O, and network I/O resources to obtain metrics such as the utilization, saturation, and number of errors for identifying system bottlenecks. Then, the tool provides tuning suggestions based on existing reference values and experience.

The tool checks the hardware, system, and component configurations for big data, distributed storage, and database scenarios, displays the configurations that are not optimal, and provides typical hardware configuration and software version information.

Prerequisites

A project has been created, and the project does not contain a node in Offline state.

Procedure

  1. Click next to the target project in the project list.

    The New Analysis Task page is displayed.

  2. Set task parameters by referring to Task Management and Table 1.

    Figure 1 shows the page for creating an overall analysis task, using a project of the general scenario as an example.
    Figure 1 Creating an overall analysis task (general scenario)
    Table 1 Parameters for creating an overall analysis task

    Parameter

    Description

    Task Name

    Name of the task. The task name must meet the following requirements:

    • Contain only letters, digits, and special characters (including @#$ %^&*()[]<>._-!-+ and spaces).
    • Contain 1 to 32 characters.

    Analysis Object

    Select System.

    Analysis Type

    Select Overall Analysis.

    Sampling Time (s)

    Sampling duration, in seconds. The default value is 300. The value range is 2 to 300. If the value exceeds the range, a message is displayed, and a backfill is automatically performed after the focus is lost.

    Sampling Interval (s)

    Sampling interval, in seconds. The default value is 1. The value range is 1 to 10.

    NOTE:

    The Sampling Interval value must be less than or equal to half of the Sampling Time value.

    (Optional) Big Data Test Tool Path

    This parameter is required only for projects of the big data scenario.

    (Optional) Distributed Storage Component Configuration File Path

    This parameter is required only for projects of the distributed storage scenario.

    Database Connection IP Address

    This parameter needs to be configured only for projects created in the Database scenario.

    (Optional) Port

    This parameter needs to be configured only for projects created in the Database scenario.

    User Name

    This parameter needs to be configured only for projects created in the Database scenario.

    Password

    This parameter needs to be configured only for projects created in the Database scenario.

    (Optional) Collecting Tracing Data

    This parameter needs to be configured only for projects created in the Database scenario. This option is not selected by default.

    Collected Data Format

    This parameter needs to be set only for projects created in the Database scenario when Collect Tracing Data is selected.

    Collect top active processes

    Whether to run the top command to collect active processes. This parameter is selected by default.

    Scheduled Startup

    Whether to create a scheduled analysis task. For details, see Task Management.

    This parameter is unavailable in the Database scenario.

    Start Now

    Whether to automatically execute a task after it is created. This parameter is selected by default.

    This parameter is unavailable in the Database scenario.

    NOTE:

    This parameter is not displayed if Scheduled Startup is enabled.
