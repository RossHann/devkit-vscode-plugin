Creating a Process/Thread Performance Analysis Task

Function Description

By using the utilization, saturation, and errors (USE) method, the tool collects information about the resources such as the CPU, memory, and storage I/O resources used by processes or threads to obtain metrics, such as the utilization, saturation, and errors and identify system bottlenecks. Then, the tool provides tuning suggestions based on existing reference values and experience. The system invoking status of a single process can be analyzed.

If the sysstat version is not later than 11.0.0, the process/thread analysis task result may contain abnormal data.

Prerequisites

A project has been created, and the project does not contain a node in Offline state.

Procedure

  1. Click next to the target project in the project list.

    The New Analysis Task page is displayed.

  2. Set task parameters by referring to Task Management and Table 1.

    Figure 1 shows the page for creating a process/thread performance analysis task, using a project of the general scenario as an example.
    Figure 1 Creating a process/thread performance analysis task (general scenario)
    Table 1 Parameters for creating a process/thread performance analysis task

    Parameter

    Description

    Task Name

    Name of the task. The task name must meet the following requirements:

    • Contain only letters, digits, and special characters (including @#$ %^&*()[]<>._-!-+ and spaces).
    • Contain 1 to 32 characters.

    Analysis Object

    Select Systems or Application.

    Mode

    Select Launch Application or Attach to Process.

    This parameter is displayed when Analysis Object is set to Application.

    Application Path

    Enter the absolute path of the application to be analyzed. For example, to analyze the loop_test application stored in the /home/test directory, enter /home/test/loop_test.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Launch Application.

    NOTE:
    • By default, applications in the /opt/ or /home/ directory are analyzed. The administrator can click in the upper right corner on the home page of the System Profiler and choose System Settings, and enter application paths (separated by semicolons) in the Application Path test box. Only administrators can modify this parameter. Common users can only view this parameter.

      You are advised to set the application path to a path such as /home or /opt. Do not set the application path to a system directory such as /, /dev, /sys, or /boot. Otherwise, system exceptions may occur.

    • The OS running user (launcher) of the Kunpeng Hyper Tuner must have the read and execute permissions on the applications to be analyzed.
    • In the multi-node scenario, you can enable Configure Node Parameters to configure this parameter separately for each node.

    (Optional) Application User

    This parameter is available when Analysis Object is set to Application and Mode is set to Launch Application. By default, this parameter is disabled.

    • Disabled: The application user is the default system user launcher.
    • Enabled: The application user is the configured user.

    (Optional) Application Parameters

    Enter application parameters based on the actual scenario.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Launch Application.

    NOTE:

    You can enable Configure Node Parameters to configure this parameter separately for each node.

    Process

    Enter a process name. The process name can be a regular expression.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Attach to Process.

    PID

    Enter a maximum of 10 PIDs of the processes to be analyzed. Use commas (,) to separate the PIDs. PID and Process can be configured at the same time.

    This parameter is displayed when Analysis Object is set to Application and Mode is set to Attach to Process.

    NOTE:
    • If Attach to Process is selected, the tool associates the ID of the process that runs an application to trace and collect the performance data of the application in real time. The OS running user launcher of the Kunpeng Hyper Tuner must have the read permission on the application.
    • To query the PID, run the ps -ef | grep Program name command.
    • You can enable Configure Node Parameters to configure this parameter separately for each node.

    Analysis Type

    Select Process/Thread Performance.

    Sampling Time (s)

    Sampling duration, in seconds. The default value is 60. The value range is 2 to 300.

    Sampling Interval (s)

    Sampling interval, in seconds. The default value is 1. The value range is 1 to 10.

    NOTE:

    The Sampling Interval value must be less than or equal to half of the Sampling Time value.

    Sampling Type

    Type of the analysis to be performed. The values are as follows:

    • CPU
    • Memory
    • Storage I/O
    • Context Switch

    Thread Collection

    Whether to collect thread information. By default, this parameter is enabled.

    System Call Tracing

    Whether to collect information about system function invoking of applications in the Linux OS. This function is disabled by default.

    This parameter is displayed when Analysis Object is set to Application.

    NOTE:

    System call tracing is not recommended for production environments because it greatly reduces the system performance when some applications are frequently called.

    Scheduled Startup

    Whether to create a scheduled analysis task. For details, see Task Management.

    If Application User is set, a scheduled startup task cannot be created. In this case, this function is unavailable.

    Start Now

    Whether to automatically execute a task after it is created. This parameter is selected by default.

    If Application User is set, a task must be executed immediately. In this case, this option is forcibly selected and dimmed.

    NOTE:

    This parameter is not displayed if Scheduled Startup is enabled.
