Ubuntu操作系统下升级失败

现象描述

V2.2.T3之后的版本在Ubuntu操作系统下升级失败。

可能原因

升级脚本Upgrade中没有HOME=/root环境变量。

处理步骤

  1. 执行如下命令进入保存鲲鹏性能分析工具新安装包的目录（“/home”请根据实际情况替换）。

    cd /home

  2. 执行如下命令解压安装包（其中“x.x.x”表示版本号，请根据实际情况替换）。

    tar -zxvf Hyper-Tuner-x.x.x.tar.gz

  3. 执行如下命令进入解压后的安装包目录。

    cd Hyper_tuner

  4. 执行如下命令再次解压安装包（其中“x.x.x”表示版本号，请根据实际情况替换）。

    tar -zxvf Hyper-Tuner-x.x.x.tar.gz

  5. 执行如下命令进入再次解压后的安装包目录。

    cd Hyper_tuner

  6. 执行如下命令获取新版本包中的升级入口功能。

    cp –rf ./Upgrade /opt/hyper_tuner

  7. 执行如下命令进入鲲鹏性能分析工具旧版本安装目录（“/opt/hyper_tuner”为工具旧版本安装目录，请根据实际情况替换）。

    cd /opt/hyper_tuner

  8. 执行如下命令升级工具（“/home”请根据实际情况替换）。

    ./Upgrade upgrade /home/Hyper_tuner/Hyper-Tuner-x.x.x.tar.gz
