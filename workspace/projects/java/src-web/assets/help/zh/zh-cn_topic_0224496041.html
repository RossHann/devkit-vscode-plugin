创建在线分析任务

前提条件

  • 已成功添加目标环境且目标环境处于在线状态，并且当前目标环境下不存在正在分析的进程。
  • 运行Java进程的JDK版本要求为JDK 8及以上。

    如果分析辅助软件是用操作系统管理员用户部署的，想对其中非操作系统管理员用户的Java进程进行在线分析，需要确保运行Java进程的JDK版本为JDK 8及以上，并在非交互shell（non-interactive shell）中配置JAVA_HOME环境变量，详细操作请参见“常用操作 > 配置JDK”。

操作步骤

  1. 登录Java性能分析Web界面，在首页界面“目标环境列表”区域选择指定目标环境，在“Java进程列表”区域选择指定的Java进程，如图1所示。

    图1 选择Java进程

    点击按钮，可以导入之前保存下载过的报告，并在导入的时候进行校验。

  2. 单击“在线分析”。

    默认打开“概览”页面，如图2所示。

    进行在线分析的进程，性能会受到较大影响。

    图2 概览
    创建在线分析任务后，点击页面右上方的按钮可获取不同的功能。
    • 点击按钮，进入“实时数据限定”配置，可以修改实时数据限定范围。
      • 点击“修改”可以设置数据限定范围。管理员用户（tunadmin）和普通用户有修改和查看权限。
      • 在Java性能分析首页中的菜单选项中选择“实时数据限定”也可以修改这项配置。
    • 点击按钮，可以查看所有页签的优化建议汇总。
    • 点击“导出报告”，勾选不同的页签可以导出数据分析报告。
    • 点击“数据清除”，在下拉框选择“全部页签”或“当前页签”，可以清除已分析的数据。

  3. 单击进程名称右边的“停止分析”可以停止分析任务，如图3所示。

    图3 停止分析任务
    停止分析任务后，单击进程名称右边的“重新开始”重启分析任务，如图4所示。
    图4 重新开始分析任务

    重新开始在线分析后，已分析的数据会被清除，请谨慎操作。
