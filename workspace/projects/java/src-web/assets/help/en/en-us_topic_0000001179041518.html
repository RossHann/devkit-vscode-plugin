Viewing Method Sampling Analysis Results

Prerequisites

A sampling analysis task is created and complete.

Procedure

  1. On the home page of the Java Profiler WebUI, click the target task name in the Sampling Analysis Records area.

    Click the CPU tab.

  2. Click the Method Sampling tab.

    View the method flame graph and call tree, Java method sample is used as an example, as shown in Figure 1 and Figure 2.
    • You can select one of the following options from the Graph Type drop-down list:
      • Flame graph: View the flame graph. When you hover the mouse pointer over a function block, the detailed information about the function is displayed.

        The Y axis indicates the call stack. Each layer has a function. A higher flame indicates a deeper stack. On the top of a flame is the function being executed, under which are its parent functions. The X axis shows the number of samplings of each function. If a function occupies a wider area on the X axis, it is more frequently sampled and its execution time is longer. Note that the X axis does not represent time. Instead, it represents all call stacks arranged in alphabetical order.

      • Call tree by thread displays the logical process of the call stack in a tree structure and analyzes the call relationship and proportion of functions in the program.
    • Sample Data contains Java method sample and Native method sample.
