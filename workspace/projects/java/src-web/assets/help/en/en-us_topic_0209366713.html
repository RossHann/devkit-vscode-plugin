Viewing Memory Analysis Results

Prerequisites

A sampling analysis task is created and complete.

Procedure

  1. On the home page of the Java Profiler WebUI, click the target task name in the Sampling Analysis Records area.
  2. Click the Memory tab.

    Click the All Objects tab. The allocation of Java objects in the heap is displayed, as shown in Figure 1. For details about the parameters, see Table 1.

    In the Classes area, click a class name to view the memory allocation bar chart and call stack of the class. Click to view all call stacks.

    Figure 1 Memory analysis (all objects)
    Table 1 Parameters on the All Objects tab page

    Parameter

    Description

    Class

    Class name.

    Max. Real Time Count

    Maximum number of real-time class instances.

    Max. Real Time Size

    Maximum memory usage of the real-time class instances.

    Memory Allocated (MiB)

    Memory allocated to the class.

  3. Click the Old-Generation Objects tab.

    The page shown in Figure Memory analysis (old-generation objects) is displayed. For details about the parameters, see Table Parameters on the Old-Generation Objects tab page.
    • You can select Stack Trace or Reference Link from the Stack Trace drop-down list on the right of the page.
    • You can enter an object name in the search box to quickly filter data.
    • You can click to view the tuning suggestions on the current task.
    Figure 2 Memory analysis (old-generation objects)
    Table 2 Parameters on the Old-Generation Objects tab page

    Parameter

    Description

    Time

    Time when the old-generation object is allocated.

    Object

    Class name.

    Thread

    Thread name.

    Used Heap

    Size of the used heap.

    Stack Trace

    Stack trace information.

    Reference Link

    Reference links.
