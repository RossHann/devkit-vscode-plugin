Creating a Profiling Task

Prerequisites

  • A Guardian is added and is in online state, and no analysis process is running in the Guardian.
  • The JDK must be JDK 8 or later.

    If the Guardian was deployed by an OS administrator and you want to perform profiling on a Java process that was not created by an OS administrator, ensure that the JDK running the Java process is JDK 8 or later, and configure the JAVA_HOME environment variable in the non-interactive shell. For details, see "Common Operations > Configuring JDK".

Procedure

  1. On the home page of the Java Profiler WebUI, select the target Guardian in the Guardian List area and select the target Java process in the Java Processes List area, as shown in Figure 1.

    Figure 1 Selecting a Java process

    Click to import the downloaded report and perform verification during the import.

  2. Click Profiling Analysis.

    The Overview tab page is displayed, as shown in Figure 2.

    The profiling process greatly affects the performance.

    Figure 2 Overview tab page
    After a profiling task is created, you can click buttons in the upper right corner of the page to perform corresponding operations.
    • Click to configure the real-time data limits.
      • You can click Modify to change the data range. The administrator tunadmin and common users have the modification and view permissions.
      • You can click on the Java Profiler home page to select the menu option about the real-time data limits to modify this parameter.
    • Click to view the tuning suggestions on all tab pages.
    • Click Export Report and select tabs to export data analysis reports.
    • Click Clear Data and select All or Current Page from the drop-down list to clear the analyzed data.

  3. Click Stop Analysis to the right of the process name to stop the analysis task, as shown in Figure 3.

    Figure 3 Stopping a profiling task
    After the profiling task is stopped, you can click Restart to the right of the process name to restart the analysis task, as shown in Figure 4.
    Figure 4 Restarting a profiling task

    Restarting a profiling task will clear the analyzed data. Exercise caution when performing this operation.
