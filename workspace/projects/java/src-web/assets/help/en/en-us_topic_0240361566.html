Run Log Management

Prerequisites

You have logged in to the Java Profiler WebUI.

Only the administrator tunadmin can change the level of run logs, and view and download run logs.

Procedure

  1. Click in the upper right corner and choose Log.

    The Operation Logs page is displayed.

  2. Click the Run Log tab.

    The Run Log tab page is displayed, as shown in Figure 1.

    When hyper-tuner-java-profiler.log reaches 100 MB, a backup named hyper-tuner-java-profiler.log.1.zip will be automatically created and saved in the /opt/hyper_tuner/thor-server-x.x.x/log/ directory. /opt/hyper_tuner is the tool installation directory, which can be changed during tool installation. x.x.x indicates the version. The original hyper-tuner-java-profiler.log.1.zip will be automatically renamed hyper-tuner-java-profiler.log.2.zip. A maximum of two compressed packages can be saved. When a third package is generated, the earliest package will be deleted by default.

    Figure 1 Run Log tab page

  3. Locate the log file to be downloaded, and click Download in the Operation column.

    In the dialog box displayed, click OK.

    The downloaded run logs are in the following format:

    Time Log level Process ID Thread name Class name Method name (line number) Log details

    Example:

    2020-07-27 09:10:08.782  WARN 20085 --- [           main]  com.zaxxer.hikari.util.DriverDataSource.<init>(70) : Registered driver with driverClassName=org.apache.derby.jdbc.EmbeddedDriver was not found, trying direct instantiation.

    The log level indicates the importance of the log information. User management run logs are classified into five levels: DEBUG, INFO, WARNING, ERROR and CRITICAL. Java profiling run logs are classified into four levels: DEBUG, INFO, WARNING, and ERROR. The default run log level is WARNING. The tool records only the logs of the specified level and higher levels.

    • DEBUG: debugging information for fault locating.
    • INFO: key information about the normal running of the service.
    • WARNING: events about the system in unexpected status that does not affect the running of the system.
    • ERROR: errors that do not affect the application running.
    • CRITICAL: errors that may cause system breakdown.
    • The administrator can click in the upper right corner on the home page of the Kunpeng Hyper Tuner or Java Profiler and choose System Settings to change the value of Run Log Level in the Common Settings area.
    • The administrator can click in the upper right corner on the home page of the Java Profiler and choose System Settings to change the value of Run Log Level in the Java Profiler Settings area.
