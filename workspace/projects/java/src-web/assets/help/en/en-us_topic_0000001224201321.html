Managing Sampling Analysis Tasks

Prerequisites

A sampling analysis task has been created and started.

Managing Sampling Analysis Tasks

On the Java Profiler WebUI, view records in the Sampling Analysis Records area, as shown in Figure 1.

Figure 1 Sampling Analysis Records area

Click next to Sampling Analysis Records to select and import a sampling analysis record file.

Click next to a profiling task to export the online analysis record file and save it to the local PC.

Click next to an analysis task to delete a sampling analysis record file.

Deleting an analysis task will also delete the historical data of the task. Exercise caution when performing this operation.

Configuring Sampling Report Thresholds

The administrator can modify the sampling report threshold, and common users can only view the threshold.

  1. Click in the upper right corner and select Report Thresholds from the drop-down list.

    The Report Thresholds page is displayed, as shown in Figure 2.

    Figure 2 Report Thresholds

  2. Click Modify Configurations.

    Change the report warning threshold and maximum report threshold.

    The number of sampling analysis reports is limited. By default, a warning message is displayed when the number of sampling analysis reports is greater than or equal to 8, and a message prompting you to delete old reports is displayed when this number is greater than or equal to 15. A maximum of 20 sampling reports can be configured. When the number of sampling reports reaches the maximum threshold, old records are not deleted and new tasks cannot be created.

    Figure 3 shows the message displayed when the number of historical reports reaches the value of Warning Threshold. Figure 4 shows the message displayed when this number reaches the value of Maximum Reports Threshold.
