Internal Communication Certificate Management

Internal communication certificates are used for communication between internal modules of the Java Profiler.

Prerequisites

You have logged in to the System Profiler WebUI.

Only the administrator tunadmin can refresh the working key. Common users can only view internal communication certificate information.

Viewing Internal Communication Certificate Information

Internal communication certificates are used for TLS communication between the server and Guardians and between the server and agents.

  1. Click in the upper right corner and choose Internal Communication Certificates.

    The page shown in Figure 1 is displayed. For details about the parameters, see Table 1.
    Figure 1 Internal communication certificates
    Table 1 Parameters on the Internal Communication Certificates page

    Parameter

    Description

    Certificate Name

    Name of the certificate.

    Status

    Certificate status. The values are as follows:

    • Valid: The remaining validity period is greater than the certificate expiration alarm threshold.
    • About to expire: The remaining validity period is less than or equal to the certificate expiration alarm threshold.
    • Expired: The certificate has expired.
    NOTE:
    • The tool automatically checks and updates the certificate status at 02:00 every day. If the current certificate is about to expire or has expired, the tool automatically updates the internal communication certificate.
    • By default, the expiration alarm threshold for internal communication certificates is 90 days. The administrator can click in the upper right corner of the home page of the Java Profiler and choose System Settings to set Internal Communication Certificate Expiration Alarm Threshold (days). The value ranges from 7 to 180 days.

    Certificate Expiration Time

    Time when the certificate will expire.

    Certificate Type

    Type of the certificate.

Refreshing the Working Key

Do not refresh the working key during the certificate update process.

The working key is used to encrypt the data to be protected in the software. For security purposes, refresh the working key periodically.

  1. Click in the upper right corner and choose Working Key.
  2. Click Refresh.

    The working key can be refreshed only once a day.
