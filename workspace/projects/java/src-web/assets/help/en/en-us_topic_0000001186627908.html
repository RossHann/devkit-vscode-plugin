The Device Is Displayed as Offline After Kunpeng Hyper Tuner Is Installed in Ubuntu

Symptom

When adding the Ubuntu 20.04.3 system node after Kunpeng Hyper Tuner is installed, the node is in the offline status.

Possible Causes

The whitelist fails to be added because the no_proxy environment variable is not configured on the server.

Procedure

  1. Configure the no_proxy environment variable in the /etc/profile file on the server.

    vi /etc/profile
    export no_proxy=127.0.0.1

  2. Make the configuration take effect.

    source /etc/profile
