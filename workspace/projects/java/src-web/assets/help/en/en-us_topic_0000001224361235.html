Java Profiler Home Page

Figure 2 shows the home page of the Java Profiler. Table 2 describes the areas on the home page.

Figure 2 Java Profiler home page
Table 2 Areas on the home page

Area

Element

Description

1

Guardian List area

Displays all Guardians added in the Java Profiler, including physical environments and container environments.

2

Java Process List area

Displays all Java processes in the current Guardian, including host processes and container processes.

3

Profiling Records area

Displays profiling records.

4

Sampling Analysis Records area

Displays sampling analysis records.

5

Current user

Displays the current login user, and provides an entry for changing the user password and logging out.

6

Back to Home

Returns to the home page of the Kunpeng Hyper Tuner.

NOTE:

You can also click the tool icon in the upper left corner of the page to return to the home page.

7

Settings

Provides navigation paths to user management, weak password dictionary management, working key management, system setting, report threshold management, real-time data restriction management, log management, and internal communication certificate management.

8

More

Provides an entry for changing the language between simplified Chinese and English, providing feedback, wizard, online help, disclaimer, and Java Profiler release information.

9

Workspace/Drive alarm area

Displays workspace and drive space information when the remaining workspace or drive space is less than 5% of the total workspace. The information is not displayed when the remaining workspace or drive space is larger than 20% of the total space.

Red: indicates a critical alarm. In this case, the sampling analysis function is unavailable. To prevent negative impacts on the tool, delete historical analysis records or release drive space.

Yellow: indicates a warning. In this case, the workspace or drive space is consumed fast and may affect the use of the tool. To prevent negative impacts on the tool, delete historical analysis records or release drive space.

NOTE:

The user's workspace is the tool installation directory. The minimum workspace recommended is 30 GB if only the Java Profiler is installed and 80 GB if the System Profiler and Java Profiler are installed.

10

Case links

Provides links to typical cases.

11

Filter box

Allows exact search by process ID and filtering by Java main class name or JAR name when there are too many process lists.

12

Data List

Displays analysis records saved or imported by users.

13

Feedback

Navigation entry for feedback.
