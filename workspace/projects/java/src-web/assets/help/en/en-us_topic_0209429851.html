Viewing Web Information

Prerequisites

A profiling task is being performed.

Viewing HTTP Requests

  1. On the home page of the Java Profiler WebUI, click the target task name in the Profiling Records area.

    The Overview tab page is displayed.

  2. Click Web and select HTTP Request.
  3. Click Start Analysis.

    The HTTP request statistics report is generated, as shown in Figure 1. For details about the parameters, see Table 1.
    • The default threshold is 50 ms. Only those operations whose duration exceeds the threshold are analyzed. The threshold value ranges from 0 ms to 10,000 ms.
    • You can click in the upper right corner of the page to save the snapshot of the current page to the Snapshot tab page.
    Figure 1 HTTP Requests tab page
    Table 1 Parameters on the HTTP Requests tab page

    Parameter

    Description

    Hot Spots

    Hot URL

    Hotspot HTTP requests (URI + HTTP request method)

    Count

    Number of times that this type of HTTP requests are executed.

    Average

    Average time (in milliseconds) for executing this type of HTTP requests.

    Real-Time Monitoring

    Executed Requests

    Number of times that the request is executed.

    Average Execution Time

    Average execute time of the request.

Viewing Spring Boot

  1. On the home page of the Java Profiler WebUI, click the target task name in the Profiling Records area.

    The Overview tab page is displayed.

  2. Click Web and select Spring Boot.

    If a login user has been configured, enter the user name and password for logging in to Spring Boot and click LOG IN.

  3. Click Start Analysis.

    The App Health Status tab page is displayed by default, as shown in Figure 2. For details about the parameters, see Table 2.

    • The Instance area displays the database name and version.
    • UP: indicates that the current process instance is running properly.
    • DOWN: indicates that the current process instance is abnormal.
    Figure 2 App Health Status tab page
    Table 2 Parameters on the App Health Status tab page

    Parameter

    Description

    Drive Space

    Total capacity

    Total drive capacity of the server.

    Available space

    Available drive space of the server.

    Threshold capacity

    Drive space threshold.

    Version

    Database version.

  4. Click the Bean Component Info tab.

    The page shown in Figure Bean Component Info tab page is displayed. For details about the parameters, see Table Parameters on the Bean Component Info tab page.
    Figure 3 Bean Component Info tab page
    Table 3 Parameters on the Bean Component Info tab page

    Parameter

    Description

    Source

    Component source.

    Dependency

    Dependency of the component.

  5. Click the Metrics Change tab.

    View the metrics change diagram, as shown in Figure 4. Table 4 describes the parameters.

    This function is applied to Spring Boot 2.x.

    Figure 4 Metrics Change tab page
    Table 4 Parameters on the Metrics Change tab page

    Parameter

    Description

    tomcat.session (Session Count)

    created

    Number of sessions created in the Tomcat container.

    expired

    Number of sessions that have expired in the Tomcat container.

    rejected

    Number of sessions rejected in the Tomcat container.

    tomcat.session (Current Session Value)

    active.current

    Number of active sessions in the Tomcat container.

    active.max

    Maximum number of active sessions in the Tomcat container.

    jvm.buffer (Buffer Count)

    count

    Count of buffers in the JVM.

    jvm.buffer (MiB)

    total.capacity

    Total capacity of the buffers in the JVM.

    memory.used

    Used memory capacity of the JVM buffers.

    logback.events.level (Event Count)

    info

    Number of info events recorded in logs.

    warn

    Number of warn events recorded in logs.

    trace

    Number of trace events recorded in logs.

    debug

    Number of debug events recorded in logs.

    NOTE:

    The number of debug events is large and is not displayed in the chart by default. You can click the corresponding color icon to display the broken line of the value.

    error

    Number of error events recorded in logs.

  6. Click the Hotspot HTTP Traces tab.

    View the HTTP trace statistical report, as shown in Hotspot HTTP Traces. Parameters on the Hotspot HTTP Traces tab page describes the parameters.
    • The default threshold is 10 ms. After the threshold is set, the system captures only the hotspot HTTP trace requests that exceed the threshold for analysis. The threshold ranges from 0 ms to 10,000 ms. After resetting the threshold, restart the HTTP trace request analysis.
    • You can enter an address in the search box for filtering.
    Figure 5 Hotspot HTTP Traces tab page
    Table 5 Parameters on the Hotspot HTTP Traces tab page

    Parameter

    Description

    Filter path

    HTTP trace filtering path at the current point of time.

    Total requests

    Total number of HTTP trace requests at the current time point.

    Successful

    Number of successful HTTP trace requests at the current time point.

    Status 4XX

    Number of HTTP trace requests in the 4XX state at the current time point.

    Status 5XX

    Number of HTTP trace requests in the 5XX state at the current time point.

    Maximum duration

    Maximum HTTP trace duration (in ms).

    Timestamp

    Time when the session occurred.

    Session ID

    Session ID.

    Method

    HPPT trace request method.

    Path

    Access path of the HTTP trace.

    Status

    Status of the HTTP trace request.

    Content-Type

    Content and type of the HTTP trace request.

    Time Used (ms)

    Time consumed by the HTTP trace request.
