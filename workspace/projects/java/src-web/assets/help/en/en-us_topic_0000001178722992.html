Configuring the Docker Container Environment

Prerequisites

  • The Docker container has been installed on the server.
  • You have obtained the OS image file.

Procedure

The following procedure uses openEuler as an example. The procedure for CentOS 7.6 is the same.

  1. Use an SSH remote login tool to upload the OS image file to any directory on the server, for example, the root directory.
  2. Use an SSH remote login tool to log in to the Linux CLI as the system user.

    This step must be performed as the root user. If you log in as a common system user, run the su command to switch to the root user and perform subsequent operations.

  3. Run the following command to mount the ISO image file to a local directory (root is the path for storing the image file and needs to be replaced with the actual path):

    mount /root/openEuler-20.03-LTS-everything-aarch64-dvd.iso /mnt

    After restart, mount the OS image file again.

  4. Run the following command to create a Docker container:

    • Host mode

      docker run -d --name containername -v /mnt/:/mnt/ --net=host --privileged=true imageid /usr/sbin/init

    • Port mapping mode

      docker run -d --name containername -v /mnt/:/mnt/ -p 8086:8086 --net=host --privileged=true imageid /usr/sbin/init

      • containername specifies the Docker container name.
      • The first /mnt/ specifies the directory to which the operating system image of the host machine is mounted in 3. Replace it with the actual directory.
      • The second /mnt/ specifies the directory of the mapped container.
      • imageid specifies the system image ID. You can run the docker images command on the host machine to query it.
      • In the port mapping mode, 8086 is the default HTTPS port of the Kunpeng Hyper Tuner. Enter the mapped container port during the tool installation.

  5. Run the following command to access the Docker container:

    docker exec -it Container ID /bin/bash

    You can run the docker ps command on the host machine to query the container ID.

  6. Configure the Yum source for the Docker container.

    1. Run the following commands to back up the original Yum source. In the mv command, path indicates the backup directory. Replace it with the actual backup directory.

      cd /etc/yum.repos.d// Access the Yum source configuration directory.

      mv *.repo path// Back up the original Yum source configuration.

    2. Configure the Yum source.
      1. Run the following command to create the local.repo file:

        touch local.repo

      2. Run the following command to open the local.repo file:

        vi local.repo

      3. Press i to enter the edit mode and add the following information to the file:
        [local]
name=local repo
baseurl=file:///mnt
enabled=1
gpgcheck=0
      4. Press Esc, type :wq, and press Enter to save the settings and exit.
    3. Update the Yum source:

      yum clean all

      yum makecache

  7. Install the dependencies:

    • CentOS:

      yum install shadow passwd vim tar file

    • openEuler:

      yum install iproute shadow passwd vim tar

  8. Open the /etc/hosts file:

    vi /etc/hosts

    Add the following content to the file (Replace Docker IP with the actual container IP address. You can run the ip addr command on the Docker container to query the IP address):

    Docker IP address  Malluma
