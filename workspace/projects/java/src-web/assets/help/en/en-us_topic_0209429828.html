Viewing Thread Information

Prerequisites

A profiling task is being performed.

Procedure

  1. On the home page of the Java Profiler WebUI, click the target task name in the Profiling Records area.

    The Overview tab page is displayed.

  2. Click the CPU tab.

    The Threads tab page is displayed, as shown in Figure 1.
    • The Thread List tab page lists the names and status of the threads started by the Java process. The thread can be in one of the following states: Runnable, Waiting, or Blocked.
    • You can use the thread search box and Display Mode drop-down list to quickly filter data.
    • You can click the Dump button to dump the current thread status for subsequent tab page analysis and display.
    Figure 1 Thread List tab page

  3. Click the Thread Dump tab.

    By default, the Lock Analysis diagram is displayed. Click the specified thread dump to view the thread lock analysis graph, as shown in Lock analysis.
    • In the lock analysis graph, threads are displayed on the left, and lock instances are displayed on the right. The color indicates the thread status. Green indicates Runnable, red indicates Blocked, yellow indicates Waiting, and orange indicates Timed_waiting.
    • The lock analysis diagram uses lines to show the lock occupation by threads. A solid line indicates that the lock is used by the thread, and a dashed line indicates that the thread is blocked on the lock instance and waits for other threads to release the lock.
    • Observation: If it is enabled, the threads and their locks or the locks and their requested threads will be highlighted. You can also click a lock and the lock and its requesting threads are highlighted.
    • Comparison: After enabling this mode, select the thread dump to be compared. By default, the comparison results of the threads and their locks are displayed. By using the comparison mode and the observation mode together, you can better observe the status change of threads and locks at two time points. Under the Comparison mode, you can enable Observation and click a lock to observe the status change of the lock and its requesting threads at two time points. The comparison mode can be enabled only after two or more thread dumps.
    • Deadlocks can be detected in the target Java application. You can locate the cause of a deadlock based on the thread dump.
    • You can click icons in the lower right corner to zoom in or zoom out the lock analysis diagram and view the diagram in full screen.
    • You can click to save the current thread dump information. The same thread dump information can be saved only once. You can download the information from the Data List on the main page.
    Figure 2 Lock analysis

  4. Select Raw data from the drop-down list.

    You can click a thread name to view the event information about the thread, as shown in Raw data.
    • A Java thread dump is a snapshot of all threads running in the JVM at a specified time.
    • You can click to delete thread dump event information of different time points.
    • You can click to save the current thread dump information. The information about the same thread can be saved only once. You can download the information from the Data List on the main page.
    Figure 3 Raw data
