User Management

Prerequisites

You have logged in to the Kunpeng Hyper Tuner or Java Profiler WebUI.

  • Only the administrator tunadmin can create, modify, and delete users.
  • The user role specifies the operations that can be performed by the user:
    • tunadmin: default WebUI login user with the administrator role. This user can view, modify, and delete its own Guardians, view and delete the Guardians of other users, manage Java processes in its own Guardians, filter specified users for sampling analysis, and perform the delete operation. This user cannot view, modify, or delete the analysis tasks created by other users.
    • User: a common user who can create Guardians and analysis tasks, and modify or delete its own Guardians and analysis tasks. This user cannot view, modify, or delete the Guardians and analysis tasks of other users.

Procedure

  1. Click in the upper right corner and choose User Management.

    The User Management page is displayed.

  2. Click Add.

    The Add User dialog box is displayed, as shown in Figure 1. For details about the parameters, see Table 1.

    Figure 1 Adding a user
    Table 1 Parameters for adding a user

    Parameter

    Description

    Name

    Name of the user to be added. The user name must meet the following requirements:

    • Start with a letter.
    • Contain 6 to 32 characters.
    • Allow only letters, digits, hyphens (-), and underscores (_).

    Admin Password

    Password of the administrator.

    Password

    Password of the user to be added. The password must meet the following complexity requirements:

    • Contain 8 to 32 characters.
    • Contain at least two of the following character types: uppercase letters, lowercase letters, digits, and special characters (`~!@#$%^&*()-_=+\|[{}];:'",<.>/?).
    • Cannot be the same as the user name or the user name in reverse order.
    • Cannot be a password in the weak password dictionary.

    Confirm Password

    Password re-entered for confirmation.

    When a common user is created, the initial password is set. When logging in to the system upon the first time, the user must reset the password, and change this password periodically for security purposes.

  3. Set parameters according to Table 1 and click OK.

    The created user is displayed in the user list on the User Management page.

    You can click the button in the Operation column of a specified user to perform the following operations:

    • Resetting the password: Enter the administrator password and new password to reset the user password.
    • Deletion: Enter the administrator password to delete a user. After a user is deleted, all historical data of the user will be deleted. Exercise caution when performing this operation.
